遇到的问题 (7)

GCC 12.3 -Werror=infinite-recursion导致vsa_hpp及3rdparty编译失败

方案: 修改3个tool_chain.cmake文件，添加-Wno-error=infinite-recursion编译选项

来源: vsa_hpp/src/ock/cmake/tool_chain.cmake, 3rdparty/hcps/src/ock/cmake/tool_chain.cmake, 3rdparty/memory_bridge/src/ock/cmake/tool_chain.cmake

AscendCLMock acl_op.h缺少#include <string>导致std::map编译错误

方案: 在acl_op.h中添加#include <string>和#include <functional>

来源: feature_retrieval/opensource/AscendCLMock/acl/acl_op.h

CMake找不到/usr/local/AscendMiniOs路径

方案: 创建符号链接从CANN 9.0.0目录到/usr/local/AscendMiniOs

来源: feature_retrieval/src/ascendfaiss/CMakeLists.txt ASCEND_MINIOS_HOME检查

Faiss和GTest库路径不匹配(CMake期望lib/，实际安装为lib64/)

方案: 创建lib->lib64符号链接

来源: feature_retrieval/src/ascendfaiss/ut/cmake/config/env.cmake

ascendsearch库构建需要ivfsp_impl目录(git clone未完整拉取)

方案: git checkout ivfsp_impl恢复源码

来源: vstar_great_impl/mix-index/publish_union.sh

链接错误: cannot find -lascend_hal

方案: 创建符号链接从CANN devlib到AscendMiniOs/lib64

来源: CANN 9.0.0 aarch64-linux/devlib/device/libascend_hal.so

4个IVFSQT单元测试失败(mock框架虚函数地址问题)

方案: 已知限制，mockcpp框架在非真实硬件环境下无法正确模拟虚函数地址

来源: feature_retrieval/ut/cases/ivfsqt/
