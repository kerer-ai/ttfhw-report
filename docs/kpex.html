文档缺失 (5)

#1setup.py中KUPL_ROOT的library_dirs配置错误，指向include而非lib目录
#2README缺少libomp、memkind等运行时依赖说明
#3README未说明需要libsdma_dk.so（KuDNN依赖）在非加速器环境的处理方式
#4README未明确说明KBLAS需使用SVE版本（非NEON版本）
#5README中的optimization_test需要torchvision但未列出此依赖

遇到的问题 (9)

setup.py中KUPL_ROOT的library_dirs配置错误

方案: 通过LDFLAGS='-L/root/HPCKit/HPCKit/latest/kupl/bisheng/release/lib'手动指定库路径

来源: setup.py

GCC不支持-stdlib=libstdc++选项（这是Clang选项）

方案: 切换到BiSheng编译器(基于Clang): export CC=clang; export CXX=clang++

来源: setup.py CFLAGS

缺少libomp.so运行时库

方案: yum install -y libomp

来源: BiSheng编译的C++扩展依赖OpenMP运行时库

缺少libmemkind.so.0运行时库

方案: yum install -y memkind

来源: PyTorch的C++扩展依赖memkind库

缺少libsdma_dk.so运行时库（鲲鹏加速器DMA库）

方案: 创建stub库提供空符号实现

来源: KuDNN运行时依赖，在非加速器环境不存在

libkudnn.so未定义符号cblas_sbgemm

方案: 将LD_LIBRARY_PATH中的KBLAS路径从neon版本改为sve版本

来源: KuDNN依赖KBLAS的batched BLAS函数，该函数仅在SVE/SME版本的libkblas.so中提供

KuPL并行初始化malloc failed（容器内共享内存限制）

方案: 错误被KPEX正确处理，不影响功能验证。可通过--shm-size增加容器共享内存改善

来源: KuPL的ULT(用户级线程)初始化需要共享内存

PyTorch从GitHub克隆失败（RPC failed）

方案: 使用Gitee镜像仓库克隆PyTorch源码

来源: GitHub连接不稳定

缺少wheel包导致bdist_wheel错误

方案: pip install wheel

来源: pip install 构建流程
