文档缺失 (5)

#1README.md 与 install_guide.md 未明确说明无 NPU 设备时的最小可验证范围；新手在普通 ARM 服务器上无法快速发现『UT 全部失败』根因来自硬件
#2docker/OVERVIEW.zh.md 提到镜像配套 26.0.0 分支，但镜像 master tag 仍出现在文档示例 master-a3-openeuler24.03-py3.11-x86_64 中，实际 AscendHub 不可用，建议同步剔除
#3install_guide.md 手动安装章节列出 Megatron-LM 来源为 https://github.com/NVIDIA/Megatron-LM.git，对中国大陆环境访问较慢，建议补充 gitcode mirror 镜像地址（如 gh_mirrors/me/Megatron-LM）
#4.pre-commit-config.yaml 默认包含会修改文件的 fixer hook（trailing-whitespace、end-of-file-fixer），CI 文档未提示开发者在本地仅做检查时应使用 --no-fix 或临时副本
#5examples/vace/inference/inference_wan2.1_14b.json 文件本身存在 JSON 格式错误（pre-commit check-json 检测到），仓库已有 JSON 不合法，需修复

遇到的问题 (6)

AscendHub 上无 master tag 镜像

方案: 按 install_guide.md 指引改用最新商用版镜像 26.0.0-910b-openeuler24.03-py3.11-aarch64，并同步将源码切换到 26.0.0 分支保持版本一致

宿主机 CentOS 7 无 python3 / pip3，无法在宿主机执行 pre-commit

方案: 改在容器内执行（容器内 Python 3.11.14）

pre-commit 默认 fixer 自动改写源码 30+ 文件，违反『不允许修改源码』约束

方案: 立即 git checkout -- . 还原全部修改；之后改用『先 cp 源码到 /tmp/mm_lint 临时副本，再在副本上跑 pre-commit』的只读方式，避免污染挂载源码

Megatron-LM 与 MindSpeed 加速库未在镜像内 pip 注册（按文档需手动 clone）

方案: 按 install_guide.md 手动安装章节：在 /opt/deps 下分别 clone MindSpeed (gitcode 直链) 与 Megatron-LM (gitcode/gh_mirrors/me/Megatron-LM 镜像)；MindSpeed pip install -e .，Megatron-LM 通过 PYTHONPATH 加入。整个过程完全在容器内 /opt/deps 进行，未触碰 MindSpeed-MM 源码目录

Megatron-LM 主仓 https://gitcode.com/Github_Mirror/Megatron-LM.git 返回 403

方案: 改用 https://gitcode.com/gh_mirrors/me/Megatron-LM.git 镜像，clone 成功

全部 UT 在 collect 阶段失败（torch_npu 初始化报错）

方案: 确认根因为本机无 NPU 设备（无 npu-smi、无 /dev/davinci*），文档明确 MindSpeed-MM 需运行在昇腾 NPU 训练硬件之上；如实记录失败而不修改源码或绕过；建议在带 NPU 的 Atlas A2/A3 主机上重测
