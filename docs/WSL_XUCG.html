#4README.origin缺少测试运行说明 - gtest测试编译和运行方法未在文档中描述
#5GTest依赖获取方式不明确 - 默认从GitHub下载可能失败，需手动指定UCG_BUILD_WITH_GTEST
#6README未说明编译标志 mcpu=tsv110 和 march=armv8.2-a 的含义和AArch64依赖
#7stub.cpp 中有未使用变量导致 -Werror 编译失败（GCC 12.3.1 + -Wall -Werror）
#8README中系统规格未列出 openEuler 24.03 LTS 基版（仅SP3），但实际可运行
#9README未说明在不安装HPCKit的情况下如何使用系统GCC编译xucg+hucx

遇到的问题 (5)

hucx(UCX)不在openEuler系统仓库中

方案: 从源码编译hucx (git clone + autogen.sh + configure + make)

来源: README - 编译安装章节5.3

GTest通过FetchContent从GitHub下载失败 (QEMU容器网络限制)

方案: 安装系统gtest-devel并创建wrapper目录结构 (UCG_BUILD_WITH_GTEST=/gtest-wrapper)

来源: 适配系统GTest路径到项目期望的googletest目录结构

stub.cpp未使用变量导致-Werror编译失败

方案: 使用 make CXX_FLAGS 覆盖编译标志，移除 -Werror 保留其他警告

来源: GCC 12.3.1在-Wall -Werror下报告unused-variable错误

ucg_gtest allgatherv_bruck 测试段错误 (QEMU用户态仿真)

方案: 标记为QEMU环境限制，非代码缺陷 (ucs_mpool_get_grow中的内存池操作)

来源: QEMU ARM64用户态仿真对UCX零拷贝/内存池操作支持不完整

hucx编译耗时较长 (QEMU仿真下约9分钟)

方案: 使用-j8 (宿主机24核的33%，QEMU下保守值) 平衡编译速度和稳定性

来源: QEMU仿真性能限制
