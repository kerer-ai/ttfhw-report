#1README 中推荐的 dnf builddep iSulad.spec 方式在 openEuler 24.03 LTS 中因 libisula-devel 版本不满足而失败，需手动安装依赖

#4构建文档未说明 openEuler 24.03 LTS 中 libisula-devel 版本不匹配问题及解决方案
#5缺少 .devcontainer 或容器化开发环境配置，外部开发者需自行搭建完整依赖链

遇到的问题 (4)

dnf builddep iSulad.spec 失败 — libisula-devel > 2.1.5-0 在 openEuler 24.03 LTS 仓库中不存在（仅 2.1.4-13）

方案: 手动安装所有依赖包（dnf install 逐个包名），接受 libisula 2.1.4-13 版本

来源: openEuler 24.03 LTS 仓库版本滞后

cmake 配置失败 — 缺少 ncurses 包

方案: dnf install -y ncurses-devel

来源: 文档未列出该依赖

编译失败 — libisula 2.1.4 API 不兼容：isula_fopen()/isula_sub_string() 函数不存在；shim_client_process_state 结构体缺少 syslog_driver/syslog_tag/syslog_facility 字段

方案: Patch 3 个源文件：common.c (isula_fopen→fopen), terminal.c (isula_sub_string→strndup), process.c (注释 syslog 代码), isula_rt_ops.c (stub 3 个 syslog 处理函数)

来源: 源码依赖 libisula >= 2.1.5，但仓库仅有 2.1.4

isula/isulad 启动时报 'Could not set locale to en_US.UTF-8'

方案: dnf install -y glibc-locale-source && localedef -i en_US -f UTF-8 en_US.UTF-8

来源: openEuler 24.03 LTS 容器默认无 en_US.UTF-8 locale
