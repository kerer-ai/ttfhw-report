遇到的问题 (7)

dnf 安装时 GPG 检查失败（gc 包的 GPG key 不匹配）

方案: 使用 --nogpgcheck 参数跳过 GPG 检查

来源: openEuler 24.03 LTS 官方仓库 GPG key 配置问题

pip 默认 PyPI 源无法访问，无法安装 Python 包

方案: 改用清华镜像源（https://pypi.tuna.tsinghua.edu.cn/simple）

来源: 容器内网络环境限制

构建时 bdist_wheel 命令不可用

方案: pip3 install wheel（清华镜像源）

来源: build.py 中 setup.py bdist_wheel 需要 wheel 包

单元测试中 9 个 test_config.py 测试因 ASCEND_HOME_PATH 环境变量缺失而失败

方案: 需要安装 CANN 软件包并配置环境变量（属于硬件依赖，非代码问题）

来源: mskl/utils/launcher_utils.py:26 get_cann_path() 函数

单元测试中 3 个 test_kernel_modifier.py 测试因文件权限断言失败

方案: Docker 挂载的 Windows/WSL 文件系统中所有文件权限为 0o777，与测试期望的 0o640 不一致

来源: Docker 跨平台文件系统挂载权限差异

pre-commit 执行时 pre-commit-hooks 安装缺少 ruamel.yaml 依赖

方案: dnf install python3-ruamel-yaml（openEuler 原生包）
