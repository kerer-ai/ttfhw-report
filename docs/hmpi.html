文档缺失 (5)

#1README 未明确说明 Fortran 编译可能遇到的问题及解决方案
#2contrib/platform/mellanox/optimized 平台文件强制启用 oshmem-fortran，与 --disable-mpi-fortran 不兼容，文档未提及此限制
#3hucx 编译需要 libibverbs-devel 和 librdmacm-devel，但 README 未列出这些 RDMA 依赖
#4xucg 编译需要显式设置 C_INCLUDE_PATH 指向 hucx 头文件目录，文档未说明
#5缺少 aarch64 架构下的具体构建指南

遇到的问题 (5)

hucx 编译失败: infiniband/verbs.h: No such file or directory

方案: 安装 libibverbs-devel 和 librdmacm-devel 包

来源: hucx 源码中 UCX RDMA 传输层依赖

hucx 编译失败: uint32_t vs ucs_status_t 函数声明冲突

方案: 从 master 分支切换到 v26.0.RC1-huawei 稳定 tag

来源: hucx master 分支代码不稳定

xucg 编译失败: ucs/datastruct/list.h: No such file or directory

方案: 设置 C_INCLUDE_PATH=/root/install/hucx/include 和 CXX_INCLUDE_PATH=/root/install/hucx/include

来源: xucg CMake 未自动检测 hucx 头文件路径

hmpi make 失败: Fortran Derived type 'mpi_datatype' is being used before it is defined

方案: 禁用 Fortran 支持: --disable-mpi-fortran --disable-oshmem-fortran，并移除 --with-platform 参数

来源: hmpi Fortran 模块依赖顺序问题，平台文件强制启用 oshmem-fortran 导致冲突

configure 失败: --disable-mpi-fortran 与平台文件 enable_oshmem_fortran=yes 冲突

方案: 不使用 --with-platform=contrib/platform/mellanox/optimized 参数

来源: contrib/platform/mellanox/optimized 文件中硬编码了 enable_oshmem_fortran=yes
