文档缺失 (7)

#1文档推荐 Ubuntu 22.04 LTS，未提及 openEuler 适配（实际可行）
#2prometheus-cpp-lite 子模块仅指定 branch=master（无版本号），v2 重构目录结构且不再是 header-only，构建不兼容
#3spdlogger4c.h 缺少 #include <cstdint>（openEuler GCC 12.3.1 暴露）
#4prometheus/core.h 缺少 #include <charconv>（std::to_chars 不可用）
#5文档未列出完整构建依赖（python3-devel、patch、dos2unix 等需从构建失败推断）
#6GitHub 子模块在中国大陆网络不可达，需替换数据源（gitcode 镜像 + ghproxy）
#7test_ubs_io_enabled.cpp 被 test_mmc_test 的 GLOB_RECURSE 重复编译，且 ClientInit 测试依赖 MmcClientDefault 单例残留状态（测试顺序敏感）

遇到的问题 (7)

GitHub 子模块克隆超时（中国大陆网络不可达）

方案: .gitmodules 中 GitHub URL 替换为 gitcode.com/gh_mirrors/<首2字符>/<仓库名>；prometheus-cpp-lite 和 mockcpp 无镜像，从 ghproxy.net 下载 tarball

来源: .gitmodules 全部 7 个子模块

prometheus-cpp-lite v2 目录重组（include/ 替代 simpleapi/include/ + core/include/）

方案: 手动创建 simpleapi/include/prometheus/ + core/include/prometheus/，复制头文件及 ip-sockets-cpp-lite 依赖

来源: prometheus-cpp-lite master 分支 v2 重构

spdlogger4c.h:55 int32_t 未声明；prometheus/core.h:152 to_chars 不是 std 成员

方案: CXXFLAGS='-include cstdint -include charconv'

来源: 头文件缺少标准库包含

RELEASE 模式 Wcast-qual + Werror 导致 prometheus v2 ip-sockets 头文件编译失败

方案: 改用 DEBUG 模式（不使用 -Werror）

来源: CMakeLists.txt RELEASE 编译选项

链接错误：undefined reference to prometheus::global_registry

方案: 编译 prometheus-cpp-lite/src/simpleapi.cpp → simpleapi.o，注入 build.ninja 链接命令

来源: prometheus-cpp-lite v2 不再是纯 header-only

mockcpp patch 行尾 CRLF/LF 不匹配

方案: 对 mockcpp 源文件及 .patch 执行 dos2unix

来源: ghproxy 下载的 tarball 行尾格式为 Windows 风格

test_mmc_test 全量执行时套件间 TCP 端口竞争（MetaService 测试共享端口 5678 等）

方案: sysctl net.ipv4.tcp_tw_reuse=1；按 CI 链路顺序执行（TestMmcServiceInterface → … → ClientInit）

来源: 多个 MetaService 测试套件绑定相同 TCP 端口
