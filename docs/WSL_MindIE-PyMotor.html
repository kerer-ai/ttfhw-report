遇到的问题 (4)

build.sh 和 generate_proto.sh 脚本包含 Windows CRLF 换行符，导致 bash 执行时报错 '$'\r': command not found' 和 'set: pipefail: invalid option name'

方案: 使用 sed -i 's/\r$//' 移除文件中的 CR 字符

来源: 运行 bash build.sh 时的错误信息

grpcio-tools 未安装，generate_proto.sh 检查失败

方案: 通过 pip3 install grpcio-tools 安装（耗时较长，约 3-5 分钟）

来源: generate_proto.sh 第 36-40 行的检查逻辑

部分 Python 依赖（pydantic、psutil、watchdog 等）未预装在 Python 3.11-slim 镜像中

方案: 通过 pip3 install 逐个安装核心依赖

来源: 运行测试时的 ModuleNotFoundError 错误

tests/common/utils/ 收集测试时失败，缺少 watchdog、requests、PIL 模块

方案: 需要额外安装这些依赖（因时间限制未完成）

来源: pytest 收集阶段的 ImportError
