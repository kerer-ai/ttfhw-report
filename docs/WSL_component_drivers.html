方案: 按文档第1.2.2节用 HTTPS 克隆 libmcpp 到 subprojects/libmcpp（环境准备，未改源码）

来源: subprojects/libmcpp.wrap + docs/0.部件驱动开发简介.md 第1.2.2节

方案: 安装 libsqlite3-dev

来源: subprojects/libmcpp/libraries/mcdatabase/meson.build:78

meson compile 失败：lauxlib.h 缺失

方案: 安装 lua 开发头文件，并清理 builddir 重新配置

来源: subprojects/libmcpp/libraries/mcapp/src/expression/lua_expression_evaluator.cpp:34

meson compile 失败（558/1314）：drivers/internal 使用 lua_rawlen/lua_isinteger，系统 LuaJIT 5.1 无此 5.3 API

方案: 使用镜像预置的厂商定制 luajit conan 包(2.1.0.B010, 含 compat-5.3.h)，生成 luajit.pc 并注册 ld 路径后清理 builddir 重新构建

来源: drivers/internal/chip/chip_base/lua_engine.cpp:153 + drivers/internal/meson.build（注释：unidev 需 Lua 5.3+，不兼容 LuaJIT）

文档推荐的 bingo build 失败：需认证 conan 仓拉取定制 luajit/2.1.0.B012

方案: 改用镜像本地已缓存的 luajit B010（含相同 5.3-compat API）继续 meson 手动构建

来源: bingo build 输出 + conan profile.dt.ini

meson compile 失败（764/1314）：wangxun pldm 光模块驱动 power.h 中 OpticalModule_Oem 未声明

方案: 经分析为源码缺陷（缺少 #include interface_pldm/optical_module/oem.h），按'不改源码'约束未修复，判定该目标构建失败

来源: drivers/pcie_nic_card/wangxun/interface_pldm/optical_module/power.h:42（提交 5066183 引入）

test_eeprom_abi 单元测试失败：签名校验找不到信任密钥文件

方案: 经分析为运行时缺少 /opt/bmc/trust/partner/device_desc_pubkey.bin，仓库与文档均未提供，判定为环境/文档缺口；其余 85 个测试套件全部通过

来源: eeprom.cpp:570 check_integrity failed: cannot open customer key file /opt/bmc/trust/partner/device_desc_pubkey.bin
