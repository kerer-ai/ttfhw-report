遇到的问题 (4)

Python Wheel 打包失败: setup.py:32 SOURCE_DATE_EPOCH=0 与 Ubuntu 22.04 默认 wheel 0.37.1 不兼容，ZIP 不支持 1980 年之前的时间戳

方案: 升级 wheel 到 0.47.0（新版本正确处理 SOURCE_DATE_EPOCH=0）

来源: setup.py 第32行 + wheel 版本过旧

pre-commit 内部 pip install 默认走 pypi.org 下载 hook 依赖，pypi.org 不可达导致初始化卡住

方案: 设置 PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple 使 pip 走清华镜像

来源: .pre-commit-config.yaml hook 环境的 Python 依赖安装

UT 子模块 googletest/mockcpp 为空目录，GitHub 访问不稳定

方案: googletest 从 gitee.com/mirrors/googletest 克隆（2s），mockcpp 从 GitHub 克隆（41s）

来源: .gitmodules 子模块定义

UT 覆盖率脚本 run_ut.sh 中 branch 覆盖率解析 awk 语法错误（branches rate 为空）

方案: 不影响测试结果，1245 测试全部通过，属脚本健壮性缺陷

来源: script/run_ut.sh 第126行
