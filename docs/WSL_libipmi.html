#6openEuler 24.03 LTS 系统 sqlite3 包未启用 SQLITE_ENABLE_PREUPDATE_HOOK，导致链接失败，未在文档中说明此兼容性要求
#7README 中 packager manager 安装命令 (apt install libipmi-dev) 为占位符，实际仓库不通过系统包管理器分发
#8未提供独立的样例/示例程序用于验证安装成功

遇到的问题 (6)

openEuler 上 ninja-build 与 python3-ninja 包冲突

方案: 使用 dnf install meson --allowerasing 让 dnf 自动解决冲突

来源: openEuler 包管理特性

libmcpp.wrap 使用 SSH URL (git@gitcode.com:) 无法在容器中克隆

方案: 使用 git config --global url.'https://gitcode.com/'.insteadOf 'git@gitcode.com:' 将 SSH 重定向为 HTTPS

来源: 通用 CI/CD 实践

libmcpp 子项目 mcdatabase 依赖 sqlite3，但 sqlite-devel 未安装

方案: 安装 sqlite-devel (openEuler 包名，对应的 Ubuntu 包为 libsqlite3-dev)。此为传递依赖，未在 README 中记录。

来源: subprojects/libmcpp/libraries/mcdatabase/meson.build

系统 sqlite3 3.42.0 缺少 sqlite3_preupdate_* 符号（openEuler 未启用 SQLITE_ENABLE_PREUPDATE_HOOK）

方案: 从源码编译 sqlite 3.43.0，使用 CFLAGS='-DSQLITE_ENABLE_PREUPDATE_HOOK' 启用所需功能，安装到 /usr/local

来源: sqlite.org 源码; subprojects/libmcpp/libraries/mcdatabase/src/sqlite_wrapper.cpp

libmcpp 子项目 mcapp 编译 lua_expression_evaluator.cpp 需要 lauxlib.h

方案: 安装 luajit-devel。此为传递依赖，未在 README 中记录。

来源: subprojects/libmcpp/libraries/mcapp/src/expression/lua_expression_evaluator.cpp

openEuler 基础镜像未安装 time 命令

方案: 无法精确测量构建时间，使用估算值
