文档缺失 (7)

#2README.md未列出findutils依赖（openEuler基础镜像缺少find命令）
#3README.md未列出sqlite3开发库依赖（libmcpp子项目mcdatabase需要）
#4README.md未列出luajit开发库依赖（libmcpp子项目mcapp需要）
#5README.md未说明openEuler系统sqlite3可能缺少SQLITE_ENABLE_PREUPDATE_HOOK编译选项，需从源码编译
#6libmcpp.wrap使用SSH URL (git@gitcode.com)，在无SSH密钥的环境中无法自动克隆
#7README.md中缺少'示例/样例'章节，仅有Lua API代码片段

遇到的问题 (5)

openEuler容器缺少find命令

方案: dnf install -y findutils

来源: 根据编译错误日志排查

ninja-build与python3-ninja冲突（python3-ninja obsoletes ninja-build）

方案: 分步安装：先安装其他依赖（使用python3-ninja），再单独安装meson（会自动安装ninja-build并升级python3-ninja）

来源: 根据dnf包冲突错误排查

libmcpp子项目mcdatabase缺少sqlite3依赖

方案: dnf install -y sqlite-devel

来源: subprojects/libmcpp/libraries/mcdatabase/meson.build

openEuler系统sqlite3未编译SQLITE_ENABLE_PREUPDATE_HOOK，导致链接时undefined reference

方案: 从源码编译sqlite3 (3.45.2)并安装到/usr/local/sqlite，启用SQLITE_ENABLE_PREUPDATE_HOOK

来源: subprojects/libmcpp/libraries/mcdatabase/meson.build - 第20行

libmcpp子项目mcapp缺少luajit头文件(lauxlib.h)

方案: dnf install -y luajit-devel lua-devel

来源: subprojects/libmcpp/libraries/mcapp/src/expression/lua_expression_evaluator.cpp
