文档缺失 (3)

#1docs/zh/install.md 未明确说明 ci/build.sh 内部会自动 git clone torch_npu 主仓与多模块 submodule，新手开发者首次构建时可能困惑。
#2README/install.md 未提示 auditwheel repair 阶段对 libccl_dpu.so 等 CANN 集合通信运行时库的隐式依赖，纯 builder 镜像中默认缺失。
#3test/run_test.py 与 examples 未在 README 中显式声明运行所需的 NPU 硬件依赖，新手在 CPU builder 中按文档操作会在最后一步打包失败。

遇到的问题 (3)

宿主机 Python 3.6.8 不满足 pre-commit>=4.0 的 Python>=3.9 要求

方案: 记录并跳过宿主级 pre-commit 执行；建议在 Python>=3.9 环境下运行 pre-commit run --all-files。

来源: .pre-commit-config.yaml

auditwheel repair 报 'libccl_dpu.so could not be located' 导致 wheel 打包失败

方案: 未修改源码；属预装 builder 镜像缺失 CANN 集合通信组件 libccl_dpu.so。可补全 LD_LIBRARY_PATH 指向含该库的 CANN runtime 包，或在 setup.py auditwheel 调用中追加 --exclude libccl_dpu.so 规避。

来源: ci/build.sh auditwheel repair 阶段

无 NPU 硬件，UT 与 examples 无法执行

方案: 记录跳过原因；UT/样例需在带 Atlas A2/A3 NPU 的整机上执行，并先安装编译产生的 torch_npu wheel。

来源: test/run_test.py + examples/
