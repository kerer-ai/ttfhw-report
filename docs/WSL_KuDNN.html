文档缺失 (7)

#1HPCKit 安装脚本不支持 openEuler 24.03 LTS 基础版（仅支持 24.03 LTS-SP3），需手动修改 SUPPORT_OS_NAME
#2基础 openEuler 镜像缺少 findutils/xargs/getopt 等基础工具，未在 README 前置条件中说明
#3README 中缺少 cmake 和 make 的显式依赖说明（虽然构建系统使用它们）
#4GoogleTest 下载依赖网络访问 github.com，未提供国内镜像替代方案
#5HPCKit 下载链接为通用页面，未提供直接下载 URL（需手动查找）
#6缺少对非 Kunpeng 硬件上 QEMU 仿真构建的性能预期说明
#7Hyper-MPI 和 HIO 组件在 openEuler 24.03 上安装失败，未在文档中说明

遇到的问题 (6)

openEuler 24.03 (LTS) 不在 HPCKit 支持的 OS 列表中

方案: 修改 bootstrapper.conf 中 SUPPORT_OS_NAME 添加 openEuler24.03(LTS)

来源: install.sh line 777 的 check_os() 函数

基础 openEuler 镜像缺少 find/xargs/getopt 等命令

方案: dnf install -y findutils util-linux diffutils

来源: 容器执行时发现

docker pull openEuler arm64 镜像被 amd64 缓存覆盖

方案: 通过 digest (sha256:7c60c59...) 拉取 arm64 manifest 并重新 tag

来源: Docker multi-arch 行为

QEMU AArch64 仿真下编译极慢（SVE/SVE2/SME 指令模拟）

方案: 降低并行任务数到 4-14 之间，但在 QEMU 下仍需数小时。建议在实际 Kunpeng 硬件上编译。

来源: 性能分析

Hyper-MPI 安装失败：不支持的 OS

方案: Hyper-MPI 不影响 KuDNN 编译，可忽略

来源: HPCKit install 输出

HIO 安装失败：缺少 libhdf5/libnetcdf/libpnetcdf

方案: HIO 不影响 KuDNN 编译，可忽略

来源: HPCKit install 输出
