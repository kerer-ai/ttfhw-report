文档缺失 (3)

#1

硬件依赖说明不够显眼

说明
README中应更明确说明MindSpeed只能在昇腾NPU硬件上运行，普通CPU/GPU环境无法使用
#2

单元测试文档缺失

说明
README未提供单元测试执行命令，仅从tests_extend目录推断使用pytest
#3

torch_npu安装指南缺失

说明
文档未提供在没有昇腾硬件时如何验证代码基础功能的方法

遇到的问题 (4)

GitHub连接失败（SSL_ERROR_SYSCALL）

方案: 使用GitCode镜像源替代克隆Megatron-LM

来源: gitcode.com/NVIDIA/Megatron-LM.git镜像可用

numpy版本冲突（Megatron需要numpy>=2.0，MindSpeed需要numpy<=1.26.0）

方案: 强制安装numpy<=1.26.0以满足MindSpeed要求

来源: requirements.txt明确指定numpy<=1.26.0

pip下载超时（网络不稳定）

方案: 使用清华镜像源加速安装

来源: pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

torch_npu依赖缺失

方案: 无法解决，需要昇腾NPU硬件环境

来源: torch_npu是昇腾NPU的PyTorch扩展，无法在普通容器中安装
