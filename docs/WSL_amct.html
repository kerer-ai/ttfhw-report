文档缺失 (7)

#1CANN Toolkit 是硬性构建前置依赖，但在 README.md 构建说明中不够突出 - 仅在“环境要求”表格中间接提及为“CANN >= 8.5.0”
#2ccache 被 CANN cmake 系统要求（cann-cmake prepare.cmake 中 ENABLE_CCACHE=TRUE），但未在依赖文档中说明
#3requirements.txt 中指定 numpy<=1.24.0，与 Python 3.12（Ubuntu 24.04 默认版本）不兼容；版本约束需更新
#4仓库中未提供 Dockerfile 用于可复现的构建环境
#5install_graph.sh 脚本从特定 OBS 地址（ascend-cann.obs.cn-north-4.myhuaweicloud.com）下载，但未在 README 或构建指南中说明
#6build.sh 脚本在执行前不检查或验证 ASCEND_HOME_PATH，导致出现缺少 securec.h 的令人困惑的错误信息
#7样例执行的前置条件（NPU 硬件、模型下载路径）记录在各样例 README 中，但未在主 README 中汇总

遇到的问题 (7)

构建失败：找不到 securec.h（CANN Toolkit 未安装）

方案: 从 https://ascend-cann.obs.cn-north-4.myhuaweicloud.com/CANN/20260131_newest/Ascend-cann-toolkit_9.0.0_linux-x86_64.run 下载并安装 CANN Toolkit 9.0.0，使用 --quiet --force 参数

来源: docs/quick_install.md 提及 CANN 安装；CMakeLists.txt 引用 ASCEND_HOME_PATH

构建失败：protobuf 构建中出现 CCACHE_PROGRAM-NOTFOUND 错误

方案: 安装 ccache 包（apt-get install ccache）

来源: cann-cmake prepare.cmake 默认设置 ENABLE_CCACHE=TRUE，将 CMAKE_C_COMPILER_LAUNCHER 设为 ccache

单元测试失败：ModuleNotFoundError: No module named 'numpy'

方案: 安装 numpy 包；注意：requirements.txt 指定 numpy<=1.24.0 与 Python 3.12 不兼容，改装 numpy 2.4.4

来源: requirements.txt 列出 numpy>=1.19.2,<=1.24.0

单元测试失败：ModuleNotFoundError: No module named 'transformers'

方案: 安装 transformers 和 pyyaml 包

来源: requirements.txt 列出 transformers==5.5.4；deepseekv3_attention_quant_module.py 从 transformers.cache_utils 导入

单元测试失败：ModuleNotFoundError: No module named 'onnx'

方案: 安装 onnx、onnxruntime 和 protobuf 包

来源: requirements.txt 列出 onnx==1.16.1、onnxruntime==1.20.0、protobuf==3.20.3

AMCT 在工作区目录下导入失败（源码遮蔽了已安装的包）

方案: 从其他目录（cd /tmp）运行导入，避免 Python 从本地源码目录而非已安装包导入

来源: Python 模块解析优先级 - 本地 amct_pytorch/ 目录优先于 pip 安装的包

样例无法在仅 CPU 的 Docker 环境中运行（需要 NPU 硬件和 torch_npu）

方案: 改为验证包导入和 API 可用性；完整样例执行需要配备 NPU 的环境

来源: examples/amct_pytorch/*/README_CN.md 列出 torch_npu 为前置条件
