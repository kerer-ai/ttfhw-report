文档缺失 (2)

#1CANN toolkit下载链接在文档中以通用URL形式提供，需要用户自行查找具体版本
#2UT命令文档描述不够明确：docs/zh/install/build.md第41行仅说明'-u启用单元测试编译模式'，未明确说明该命令会自动编译并执行所有UT（ophost/opapi/opkernel），导致验证时误用了'--ophost --opapi -u --noexec'等组合参数，而非文档推荐的简单命令'bash build.sh -u -j${n}'

遇到的问题 (5)

首次cmake配置失败，缺少git命令

方案: 安装git包

来源: dnf install -y git

CANN toolkit安装需要接受EULA协议

方案: 使用--quiet参数自动接受协议

来源: CANN安装文档

cmake配置检查版本信息文件路径不匹配

方案: 使用build.sh脚本替代直接cmake，脚本会处理路径问题

来源: 项目构建脚本

单元测试编译缺少Python.h头文件

方案: 安装python3-devel包

来源: dnf install -y python3-devel

op_api_ut链接失败，缺少libascend_hal.so

方案: 需要NPU硬件或安装NPU驱动库，纯CPU环境无法解决

来源: op_api_ut依赖NPU硬件驱动库libascend_hal.so
