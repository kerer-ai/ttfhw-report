遇到的问题 (8)

CRLF行尾导致build.sh无法执行: 'build.sh: line 15: command not found'

方案: find /workspace -name '*.sh' -exec sed -i 's/\r$//' {} \; 以及同样的 .cmake 文件

来源: Windows git clone导致的行尾问题

grpc构建失败: file failed to create directory because: File exists - grpc源码tar包中包含名为'build'的Bazel BUILD文件，与CMake构建目录冲突

方案: 修改 cmake/open_source_build/grpc.cmake，在解压后添加 file(REMOVE "${SOURCE_DIR}/build") 删除冲突文件

来源: 在cmake/open_source_build/grpc.cmake中定位到问题

ABI不兼容链接错误: undefined reference to 'google::protobuf::Message::InitializationErrorString[abi:cxx11]()'

方案: 第三方库使用ABI=0构建，但libtorch需要ABI=1。使用 --use_cxx11_abi=1 重新构建第三方库和主项目

来源: CMakeLists.txt中的 USE_CXX11_ABI 选项

zlib头文件缺失: protobuf cmake无法找到zlib.h

方案: 手动复制 zlib.h 和 zconf.h 到 /workspace/third_party/output/zlib/include/

来源: zlib构建输出目录

初始PyTorch stub不足: fatal error: torch/torch.h: No such file or directory

方案: 下载真实 libtorch CPU 2.5.1+cpu (162MB) 从 https://download.pytorch.org/libtorch/cpu/，解压到 /opt/libtorch/，并符号链接为 torch 模块

来源: setup.py 中的 PYTORCH_INSTALL_PATH 检测逻辑

mockcpp模板重定义错误导致 executor/其他测试模块编译失败: 'PredictTypeTraits' redefinition

方案: 第三方mockcpp库的bug，非MindIE-LLM源码问题。未解决，标记为跳过

来源: third_party/output/mockcpp/include/mockcpp/ChainingMockHelper.h

样例程序需要torch_npu和Ascend NPU硬件

方案: 跳过 - 样例程序无法在无NPU的容器环境中运行

来源: examples/atb_models/README.md

测试执行时libmindie_engine.so找不到: cannot open shared object file

方案: 设置综合LD_LIBRARY_PATH，包括build tree中的所有.so目录和third_party/output/*/lib/目录

来源: 项目未正确设置RPATH
