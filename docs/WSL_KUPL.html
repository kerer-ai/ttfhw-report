遇到的问题 (7)

x86_64: kupl_memory_cpu_store_fence/load_fence 未定义 (ARM DMB指令)

方案: 需添加 x86_64 arch目录实现 mfence/lfence/sfence 或 __sync_synchronize() 回退

来源: kupl_fence.h:26-32, kupl_barrier.h:37-43

x86_64: -march=armv9-a+sme-f64f64+sve 不是有效x86架构标志

方案: 需条件编译跳过MMA或提供x86 MMA实现

来源: src/CMakeLists.txt:99-101

ARM64 GCC: sme-f64f64 特性修饰符不被 GCC 12.3.1 支持

方案: 安装 Bisheng 编译器 (HPCKit) 或 GCC >= 14；核心库51/52文件可用GCC 12.3.1编译

来源: src/CMakeLists.txt:101

ARM64 Clang: linxicore6100 CPU target 不被标准 clang 17 识别

方案: 使用 Bisheng 编译器 (鲲鹏定制clang, 支持 linxicore6100)

来源: src/CMakeLists.txt:99

--enable_mma=off 未排除 mma/kupl_mma.cpp 编译

方案: 修复 CMakeLists.txt: 将 mma 源文件用 ENABLE_KUPL_MMA 条件包裹，arch 标志移入条件块

来源: build.sh + src/CMakeLists.txt:93-102 + libkupl_source_files.cmake:13

Release模式 -fvisibility=hidden 导致UT链接失败 (内部符号不可见)

方案: 测试构建需 KUPL_BUILD_KIND=test 模式编译库以导出内部符号

来源: src/CMakeLists.txt:55-56 (visibility hidden in src mode)

QEMU user-mode下多线程同步测试挂起 (sgraph_node_egroup等)

方案: QEMU限制，非代码缺陷。需原生ARM64硬件或完整系统模拟(如qemu-system-aarch64)运行

来源: QEMU user-mode emulation limitation
