文档缺失 (5)

#1文档未明确说明 openEuler/RHEL 等非 Ubuntu 系统的安装步骤
#2docker/Dockerfile 仅支持 NVIDIA CUDA 环境，无纯 CPU 或 NPU 的 Dockerfile 变体
#3pip install -e 'python[all_cpu]' 在 openEuler 上存在 editable install 元数据冲突问题，文档未提供解决方案
#4文档未说明 sglang-grpc Rust 扩展在 openEuler 上编译需要的具体依赖（protobuf-compiler 等）
#5缺少 protoc 的安装说明（Rust grpc 扩展编译必需）

遇到的问题 (3)

首次 pip install -e 失败：缺少 Rust 编译器（cargo/rustup），sglang-grpc 扩展无法编译

方案: 安装 Rust 工具链：curl -sSf https://sh.rustup.rs | sh -s -- -y --profile minimal

来源: setup.py 中 CMakeExtension 编译 sglang-grpc Rust 扩展

sglang-grpc Rust 扩展编译失败：缺少 protoc (protobuf-compiler)

方案: dnf install -y protobuf-compiler

来源: rust/sglang-grpc build.rs: prost-build 需要 protoc

pip install -e 'python[all_cpu]' 成功但 sglang.srt 模块不完整

方案: editable install 模式下存在元数据冲突（sglang-0.5.13.post1 残留 dist-info），导致 site-packages 中仅 sglang/srt/models 子目录被复制。pip install sglang(PyPI) 可正常使用，但源码 editable install 有问题

来源: pip editable install 元数据冲突
