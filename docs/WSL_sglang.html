遇到的问题 (4)

Docker 守护进程不可用，无法启动 openEuler 容器

方案: Docker CLI 已安装（v29.6.0），但守护进程无法启动（可能需要系统重启或配置 Hyper-V/WSL2）

来源: 宿主机环境限制

项目核心依赖 NVIDIA CUDA Toolkit 和 NVIDIA GPU 硬件

方案: sgl-kernel/CMakeLists.txt 中 find_package(CUDAToolkit REQUIRED)，无法在没有 NVIDIA GPU 的环境中编译 CUDA 内核

来源: sgl-kernel/CMakeLists.txt 第28行

pre-commit 多个 hooks 在 Windows 宿主机上失败

方案: language: system 类型的 hooks 需要 python3（Windows 上为 python）；clang-format 的 .clang-format 符号链接在 Windows 上解析失败；rustfmt 需要 Rust 工具链

来源: Windows 宿主机环境限制

项目需要大量第三方依赖（DeepEP, FlashInfer, Mooncake, MSCCL++, GDRCopy）

方案: 这些依赖从 GitHub 克隆编译，需要 NVIDIA GPU 环境和 CUDA 工具链

来源: docker/Dockerfile 多个构建阶段
