遇到的问题 (5)

git safe.directory 检查阻止 cann-cmake 通过 FetchContent 克隆

方案: 配置 git config --global safe.directory '*', 允许跨用户文件操作

来源: CMakeLists.txt → cmake/fetch_cann_cmake.cmake → FetchContent_Populate

缺少 CANN Toolkit 构建依赖检查（hcomm, runtime, metadef, bisheng-compiler, asc-devkit）

方案: 从 master 分支制品仓库下载并安装 CANN Toolkit 9.1.0，所有依赖版本自动满足要求

来源: version.cmake 第13-17行和 cmake/func.cmake 第250-258行

CANN Toolkit .run 安装器的 --force 参数无法跳过 EULA 交互确认

方案: 使用 --quiet 选项静默安装，该选项自动接受 EULA 并跳过所有人机交互

来源: Ascend-cann-toolkit_*.run --help 输出："--quiet: Quiet install mode, skip human-computer interactions"

来源: build.sh 第459行：find "$ut_dir" -type f -executable
