文档缺失 (7)

#1仓库无根目录 Dockerfile，只有 .devcontainer/Dockerfile，初始探索时容易遗漏
#2快速入门文档中依赖安装章节未明确 CANN toolkit 为必要依赖，而是在编译报错可能中提及
#3install_deps.sh 不支持 openEuler（仅检测 debian 和 rhel）
#4lcov 在 openEuler 24.03 LTS 仓库中不可用，需通过 pip 安装
#5ccache 最低版本要求不一致：快速入门 4.6.1，install_deps.sh 4.8.2
#6预编译的 cpudebug 库在 openEuler 上因 GLIBCXX 版本不兼容而无法链接 C++ UT
#7pytest 在 openEuler 仓库中无 python3 入口，需通过 pip 安装才能被 python3 -m pytest 使用

遇到的问题 (6)

lcov 包在 openEuler 仓库中不存在

方案: 通过 pip3 install lcov 安装

来源: install_deps.sh 第 410 行

install_deps.sh 不支持 openEuler（仅识别 debian 和 rhel）

方案: 手动通过 dnf 安装各依赖包

来源: install_deps.sh 第 45-65 行 detect_os()

首次构建时 build.sh 找不到 CANN toolkit

方案: 从 README 提供的 OBS 镜像 URL 下载 Ascend-cann-toolkit_9.1.0_linux-x86_64.run，使用 --quiet --install 安装到 /usr/local/Ascend

来源: README.md 快速入门 - 下载安装CANN包章节

构建时 bdist_wheel 命令缺失

方案: pip3 install wheel 安装 wheel 包后重试

来源: setup.py 构建 Python wheel 需要 wheel 包

C++ UT 链接失败：libcpudebug_cceprint.so 需要 GLIBCXX_3.4.32，但 openEuler 24.03 最高提供 GLIBCXX_3.4.30

方案: 预编译的 cpudebug 库（从 OBS 下载）是在较新的 Ubuntu/GCC 环境下编译的。需等待 cpudebug 库提供 openEuler 兼容版本，或升级容器中的 libstdc++

来源: cmake/third_party/cpudebug.cmake（自动下载的预编译闭源包）

dnf 安装的 python3-pytest 无法通过 python3 -m pytest 调用

方案: pip3 install pytest 安装 pip 版本的 pytest

来源: openEuler 仓库中 python3-pytest 的打包方式
