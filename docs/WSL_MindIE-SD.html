遇到的问题 (4)

构建脚本使用 Windows CRLF 行尾格式，Linux 下执行失败

方案: 使用 dos2unix 命令转换 build/*.sh 和 tests/*.sh 脚本文件

来源: build/build_ops.sh 第12行: $'\r': command not found

CANN Toolkit 未安装，无法编译 Ascend 算子和 PyTorch 插件

方案: 从华为昇腾官网下载安装 CANN Toolkit 9.0.0，设置 ASCEND_TOOLKIT_HOME 环境变量；或使用已预装 CANN 的 DevContainer 镜像

来源: build/build_ops.sh 检测 CANN_TOOLKIT_HOME 环境变量失败

torch-npu 和 triton-ascend 包不在公共 PyPI，无法直接安装

方案: 使用华为专用 PyPI 源 https://ascend.devcloud.huaweicloud.com/pypi/simple/ 或下载离线安装包

来源: requirements.txt 第4-5行指定了这些包但未说明获取方式

mindiesd/layers/flash_attn/prompt_flash_attn.py 直接 import torch_npu，无 CPU fallback 机制

方案: 可使用 unittest.mock.MagicMock 模拟 torch_npu 模块进行无 NPU 环境下的基础开发和测试；建议优化导入结构，支持可选延迟导入

来源: mindiesd/layers/flash_attn/prompt_flash_attn.py 第16行
