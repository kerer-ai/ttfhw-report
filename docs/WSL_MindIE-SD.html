遇到的问题 (9)

Container startup with 'tail -f /dev/null' failed - 'cannot open nul for reading'

方案: Changed to 'sleep infinity' as container entrypoint

来源: Docker on Windows maps /dev/null differently

CANN toolkit install failed - 'flock: command not found'

方案: Installed util-linux package which provides flock command

来源: openEuler 24.03 minimal image does not include flock by default

pip install torch from official PyPI timed out

方案: Configured Tsinghua mirror (pypi.tuna.tsinghua.edu.cn) as global pip index

来源: Network conditions in China

torch_npu could not be installed normally - requires torch==2.9.0+cpu but we have torch==2.9.0+cu128

方案: Downloaded torch_npu wheel from Huawei OBS (pytorch_v2.9.0-7.3.0_py311.tar.gz) and installed with --no-deps

来源: Huawei OBS: https://pytorch-package.obs.cn-north-4.myhuaweicloud.com/pta/Daily/v2.9.0-7.3.0/20260214.4/

Build failed: 'ld: cannot find -lopapi' - libopapi.so missing

方案: Downloaded and installed Ascend-cann-910b-ops_8.5.1_linux-x86_64.run from 昇腾OBS repository, which provides libopapi.so

来源: https://ascend-repo.obs.cn-east-2.myhuaweicloud.com/CANN/CANN%208.5.1/

Shell scripts have Windows CRLF line endings causing bash parse errors

方案: Used dos2unix to convert all .sh files in build/, tests/, csrc/, mindiesd/ directories

来源: Files in repository edited on Windows

AscendC operator compilation (build_ops.sh) failed - NPU target compilation requires Ascend NPU compilation infrastructure

方案: Skipped AscendC build by temporarily renaming csrc/ops directory. Plugin build (build_plugin.sh) succeeded independently.

来源: build/build_ops.sh calls msopgen which requires NPU compiler toolchain

mindiesd package import fails without NPU driver - torch_npu requires libhccl.so

方案: Set TORCH_DEVICE_BACKEND_AUTOLOAD=0 to prevent automatic torch_npu loading. Unit tests work with this setting.

来源: torch_npu design - requires NPU runtime libraries

Sample program (wan_infer.py) fails with ImportError: AutoencoderKLWan not in diffusers 0.29.0

方案: Not resolved - requires newer diffusers version or NPU hardware for proper testing

来源: examples/dummy_run/wan_infer.py requires diffusers >= 0.34.0 per README, but requirements.txt specifies 0.29.0
