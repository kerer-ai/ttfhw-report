文档缺失 (6)

#1README.md 中缺少详细的构建/测试步骤说明，主要依赖 Dockerfile 和 installation.md
#2未明确说明在没有 NPU 硬件时如何运行测试（测试代码本身通过 mock 和 CPU 模式支持）
#3样例程序需要模型权重，但模型下载指引不够详细（仅提及 modelers.cn）
#4DNS 配置问题未在文档中提及（内网环境可能需要配置 DNS 服务器）
#5未明确说明 msopgen 工具的获取方式，build_ops.sh 中引用了该工具但没有说明安装步骤
#6CANN Toolkit 安装指导文档（docs/zh/installation.md）仅描述了 pip 安装方式，未提及从源码构建的完整依赖

遇到的问题 (8)

远程机器 DNS 解析失败，无法访问 gitcode.com 和 Docker Hub

方案: 手动修改 /etc/resolv.conf 添加 8.8.8.8 DNS 服务器后解决

来源: 远程机器系统配置

Docker Hub 无法从远程机器访问，ubuntu:24.04 镜像拉取失败

方案: 使用远程机器已有的 openEuler 24.03 LTS 镜像作为替代

来源: 网络限制

CANN NNAL 8.5.1 安装失败 (exit code 4)

方案: NNAL 需要 NPU 硬件驱动支持，在无 NPU 环境中安装失败是预期行为

来源: CANN NNAL 安装包日志

PyTorch 2.9.0 华为 OBS 源下载超时（2GB wheel 文件）

方案: 使用 PyPI 安装 torch 2.12.0 作为替代版本，功能测试足够

来源: 网络速度慢 (约 20-30 KB/s)

构建失败：msopgen 工具不存在于 CANN Toolkit 安装目录中

方案: msopgen 是 CANN 开发工具，可能需要额外安装 CANN 开发组件。源码构建在完整 CANN 环境中应该可行

来源: build/build_ops.sh 第34行检查 msopgen 路径

tests/run_UT_test.sh 有 17 个错误，均为 'ModuleNotFoundError: No module named torch_npu'

方案: tests/UT/run.py 使用 MagicMock mock 了 torch_npu，因此可以通过。run_UT_test.sh 中的测试文件未使用 mock 导致导入失败。这是测试框架的一致性问题

来源: tests/UT/run.py 第21-27行定义了 torch_npu mock

pip 默认超时时间 15 秒，大包下载经常超时

方案: 配置 pip 超时为 300 秒，并使用华为云镜像源 (repo.huaweicloud.com)

来源: pip 配置

容器挂载路径不匹配：仓库克隆到 /home/admin123/MindIE-SD 但容器挂载到 /root/MindIE-SD

方案: 重新创建容器，使用正确的挂载路径 -v /home/admin123/MindIE-SD:/workspace

来源: 容器配置
