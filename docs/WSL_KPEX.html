文档缺失 (9)

#1HPCKit 下载链接无法自动获取: 需注册登录鲲鹏社区, README 中未提供直接下载 URL 或备选获取方式
#2README 中未列出完整的 HPCKit 组件依赖清单: KuTACC, KUPL, HMPI, KuDNN, KML BLAS 均需在安装前知晓
#3PyTorch v2.5.0 源码编译说明在 requirements_install.md 中, 但步骤极其复杂 (需打补丁, 编译数小时), README 仅一行提及
#4KPEX C++ 源码无 HPCKit 备选编译路径: 所有 HPCKit 头文件引用均为无条件 #include, 无 #ifdef 守卫或 fallback
#5未说明 openEuler 仓库已提供 PyTorch 包 (python3-pytorch), 文档仅描述源码编译方式
#6未提供测试/验证的具体预期输出或判定标准 (仅说 import kpex 无报错)
#7examples/optimization_test.py 存在但 README 未说明如何运行样例
#8setup.py 中 HMPI_ROOT, KUTACC_ROOT, KUPL_ROOT, KUDNN_ROOT 环境变量在 README 中说明不足: 未列出默认值、可选性及缺失时的行为
#9ninja-build 和 wheel 依赖未在文档中列出 (setup.py 使用 ninja 作为编译后端)

遇到的问题 (5)

C++ 源码无条件依赖 HPCKit 头文件 (kutacc.h, kudnn.hpp, kupl.h)

方案: 无法解决 - 必须从 https://www.hikunpeng.com/developer/hpc/hpckit-download 下载安装 HPCKit (需注册登录)。HPCKit 为鲲鹏专有闭源软件

来源: HPCKit 缺失

PyTorch pip install 超时 (QEMU ARM64 模拟下网络极慢, PyPI 包约 200M+)

方案: 使用 openEuler 仓库的 python3-pytorch 2.1.0a0 替代 (dnf install)

来源: 网络限制

Python.h 缺失 → kpex.cpp 编译失败

方案: dnf install python3-devel

来源: 缺少开发包

ninja 未安装 → pip install 回退到 distutils 并报 bdist_wheel 错误

方案: dnf install ninja-build && pip install wheel

来源: 缺少构建工具

PyTorch 版本不匹配: 系统装的是 2.1.0a0, 文档要求 v2.5.0 (含 HPCKit 补丁编译)

方案: 无法确定版本差异是否影响构建 - 构建在到达 PyTorch API 调用前已被 HPCKit 头文件阻塞

来源: 版本兼容性
