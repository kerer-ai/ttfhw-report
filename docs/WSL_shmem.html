#6version.info 声明依赖 Ascend HDK >= 25.0.RC1.1，但实际 CANN 9.0.0 Toolkit 可满足所有子包版本要求

遇到的问题 (5)

首次构建失败：bisheng 编译器未在 PATH 中，cmake 报错找不到 CMAKE_C_COMPILER=bisheng

方案: source /usr/local/Ascend/cann-9.0.0/set_env.sh 设置环境变量后重试

来源: CMakeLists.txt 第14行: set(CMAKE_C_COMPILER bisheng CACHE FILEPATH 'C compiler')

单元测试 511/736 失败：全部因缺少 NPU 硬件设备

方案: 需在带昇腾 NPU（Ascend NPU）的硬件上执行单元测试。当前 WSL2 x86_64 容器无 NPU 硬件

来源: tests/unittest/main_test.cpp 中 aclInit() 和 aclrtSetDevice() 调用需要物理 NPU 设备

示例运行报 Segmentation fault：无 NPU 硬件

方案: 示例需要 NPU 设备初始化（aclrtSetDevice），仅可在带昇腾 NPU 的硬件上运行

来源: 示例代码入口调用 aclrtSetDevice 初始化 NPU 设备

Python wheel 安装失败：numpy/scipy 依赖版本约束冲突

方案: 手动安装符合版本约束的 numpy 和 scipy 后安装 wheel，或调整 requirements.txt 中的版本约束范围

来源: requirements.txt: numpy>=1.24.4,<=1.26.4, scipy==1.13.1

pre-commit hook 环境下载超时：从 gitcode.com 下载 7 个 hook repo 超时

方案: 更换网络环境或配置 gitcode.com 镜像加速；hook 配置本身正确，超时是网络问题

来源: .pre-commit-config.yaml 中 7 个 remote repo 均指向 gitcode.com
