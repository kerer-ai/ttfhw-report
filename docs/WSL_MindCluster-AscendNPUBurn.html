遇到的问题 (4)

Docker 未安装在 Windows 宿主机上，无法启动 openEuler 容器进行原生构建验证

方案: 需要在配备 Docker 的 Linux 环境中执行验证，或安装 Docker Desktop for Windows

来源: 环境限制

torch_npu（Ascend Extension for PyTorch）未安装

方案: torch_npu 需在配备 Ascend NPU 硬件的 Linux 服务器上安装，需从昇腾社区下载与 CANN 版本配套的安装包

来源: README.md 软件配套要求

PyTorch Windows 版 DLL 加载失败 (c10.dll)，无法执行 setup.py 构建

方案: 该仓库的 C++ 扩展（NpuExtension）依赖 torch_npu 的 ACL 接口，必须在 Linux + Ascend NPU 环境下构建

来源: build/setup.py 第47-48行: import torch_npu; from torch_npu.utils.cpp_extension import NpuExtension

pre-commit codespell hook 对 typos.toml 配置文件自身产生误报

方案: 应在 .pre-commit-config.yaml 的 codespell hook 中添加 exclude 规则排除 pre-commit/typos.toml，或在 typos.toml 中将误报词加入忽略列表

来源: .pre-commit-config.yaml codespell hook 配置
