遇到的问题 (8)

Windows 换行符导致 shell 脚本无法执行

方案: 使用 sed -i 's/\r$//' 修复所有 .sh 文件的行尾符

来源: 容器内执行

grpc 源码构建失败：子依赖 (abseil-cpp, protobuf, re2, c-ares, zlib) 的 CMakeLists.txt 在解压后缺失

方案: 跳过 grpc 源码编译 (build.sh --no-grpc)，改用 openEuler 系统 grpc 包 + 手动创建 cmake config 文件

来源: build/third_party/build.sh 中 grpc 子目录不完整

protobuf cmake 配置引用 libprotobuf-lite.so 但系统未提供

方案: 创建符号链接：libprotobuf-lite.so -> libprotobuf.so

来源: openEuler 24.03 protobuf-devel 包不包含 lite 库

grpc cmake 配置文件在 openEuler 包中不存在

方案: 手动创建 gRPCConfig.cmake，定义 gRPC::grpc++, gRPC::grpc++_reflection, gRPC::grpc 导入目标

来源: openEuler 24.03 grpc-devel 包不包含 cmake 配置

grpc 内部头文件 (grpc_tls_credentials_options.h) 未安装在系统 include 路径

方案: 从 grpc 源码目录创建符号链接到 install include 路径

来源: mindie_motor/src/common/cluster_grpc/GrpcClusterClient.h:16

msServiceProfiler/Tracer.h 是 CANN (Ascend) 专属头文件，openEuler 容器中不可用

方案: 无法解决 - 这是硬性 CANN 依赖。代码中使用 #ifdef WITH_PROF 但 coordinator 的 ServerConnection.h 直接 include 该头文件而不受条件编译保护

来源: coordinator/common/http_server/ServerConnection.h:27

Python 测试在导入阶段因单例初始化失败无法收集

方案: 测试需要完整 MindIE 运行环境（TLS 证书、配置文件），无法在干净容器中独立运行

来源: node_manager/framework/client.py:50 - _initialize_tls_context()

pip 安装 grpcio 耗时过长（下载大体积 wheel）

方案: 安装被中断；Python 测试不需要 grpcio 即可导入验证

来源: 网络安装
