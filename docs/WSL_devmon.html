#1README 声称需要 CMake 3.10+，但实际构建系统为 Meson+Ninja（CMake 仅用于 subproject 中的依赖查找），README 构建说明有误导性
#2README 推荐 Ubuntu 20.04，但未提供 Dockerfile 或 devcontainer 配置，缺少容器化开发环境说明
#3仓库未在文档中说明 subproject 的 git SSH URL 依赖（libmcpp、component_drivers），外部开发者可能因无 SSH 密钥而无法获取依赖
#4README 未提及 sqlite3 的特殊编译需求（libmcpp subproject 需要 SQLITE_ENABLE_PREUPDATE_HOOK），openEuler 默认 sqlite3 不满足
#5README 未说明 Lua API 版本需求——代码使用了 Lua 5.2+ 函数（lua_rawlen）和 Lua 5.3+ 函数（lua_isinteger），但依赖声明为 luajit（Lua 5.1），存在文档不一致
#6README 推荐的 apt 包名（libboost-all-dev、libgtest-dev）与 openEuler dnf 包名（boost-devel、gtest-devel）存在差异，缺少跨发行版构建说明
#7README 中的 busctl 样例命令依赖完整 BMC D-Bus 服务栈，无法在独立容器中执行，缺少最小化验证样例

遇到的问题 (5)

meson setup 阶段 libmcpp subproject 的 mcdatabase 模块找不到 sqlite3（通过 pkg-config）

方案: 通过 openEuler 仓库安装 sqlite-devel（dnf install sqlite-devel）

来源: libmcpp/libraries/mcdatabase/meson.build 第78行: dependency('sqlite3')

链接阶段 sqlite3_preupdate_hook、sqlite3_preupdate_old 等符号未定义——openEuler 的 sqlite3 编译时未开启 SQLITE_ENABLE_PREUPDATE_HOOK

方案: 从 sqlite.org 下载 sqlite-autoconf-3420000.tar.gz，使用 CFLAGS="-DSQLITE_ENABLE_PREUPDATE_HOOK=1" 从源码编译安装到 /usr/lib64

来源: libmcpp/libraries/mcdatabase/src/sqlite_wrapper.cpp

编译阶段 component_drivers subproject 报错：lua_rawlen 和 lua_isinteger 未声明——LuaJIT 2.1 仅提供 Lua 5.1 API，不支持代码使用的 Lua 5.2+ 函数

方案: 安装 lua-devel 5.4.6，创建 lua5.4.pc → lua.pc 符号链接，并将 /usr/lib64/pkgconfig/luajit.pc 替换为指向 lua 5.4 的内容（备份原文件为 luajit.pc.bak）

来源: component_drivers/drivers/internal/chip/chip_base/lua_engine.cpp 和 lua_chip_binding.cpp

subproject 通过 git@gitcode.com: SSH URL 获取，容器内无 SSH 密钥

方案: 配置 git URL 重写：git config --global url."https://gitcode.com/".insteadOf "git@gitcode.com:"

来源: subprojects/component_drivers.wrap 和 subprojects/libmcpp.wrap

1 个测试用例失败：TopologyTest.test_dump_topology_uses_parent_relation_for_sibling_paths

方案: 该测试依赖完整 BMC 环境中的 D-Bus 服务（bmc.kepler.persistence、bmc.kepler.hwdiscovery），在独立容器中无法启动这些服务。属于环境限制，非代码缺陷

来源: libs/devmon/tests/test_topology.cpp:7640

documentation_gaps
  1. #1
    README 声称需要 CMake 3.10+，但实际构建系统为 Meson+Ninja（CMake 仅用于 subproject 中的依赖查找），README 构建说明有误导性
  2. #2
    README 推荐 Ubuntu 20.04，但未提供 Dockerfile 或 devcontainer 配置，缺少容器化开发环境说明
  3. #3
    仓库未在文档中说明 subproject 的 git SSH URL 依赖（libmcpp、component_drivers），外部开发者可能因无 SSH 密钥而无法获取依赖
  4. #4
    README 未提及 sqlite3 的特殊编译需求（libmcpp subproject 需要 SQLITE_ENABLE_PREUPDATE_HOOK），openEuler 默认 sqlite3 不满足
  5. #5
    README 未说明 Lua API 版本需求——代码使用了 Lua 5.2+ 函数（lua_rawlen）和 Lua 5.3+ 函数（lua_isinteger），但依赖声明为 luajit（Lua 5.1），存在文档不一致
  6. #6
    README 推荐的 apt 包名（libboost-all-dev、libgtest-dev）与 openEuler dnf 包名（boost-devel、gtest-devel）存在差异，缺少跨发行版构建说明
  7. #7
    README 中的 busctl 样例命令依赖完整 BMC D-Bus 服务栈，无法在独立容器中执行，缺少最小化验证样例
