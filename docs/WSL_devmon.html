遇到的问题 (5)

Subproject wrap files use SSH URLs (git@gitcode.com)

方案: Configured git to use HTTPS instead: git config --global url.'https://gitcode.com/'.insteadOf 'git@gitcode.com:'

来源: Standard git configuration workaround

SQLite3 missing preupdate_hook symbols

方案: Compiled SQLite 3.47.2 from source with -DSQLITE_ENABLE_PREUPDATE_HOOK flag

来源: meson.build analysis revealed SQLITE_ENABLE_PREUPDATE_HOOK compile argument

openEuler SQLite package lacks SQLITE_ENABLE_PREUPDATE_HOOK

方案: Downloaded and compiled SQLite from sqlite.org with required flags

来源: SQLite official download page

C++ template compatibility error between libmcpp and component_drivers

方案: UNRESOLVED - This is a code-level compatibility issue in the subprojects that cannot be fixed by installing dependencies

来源: Build error at subprojects/component_drivers/drivers/csr1/chip/common/interface/i_aggregate.cpp:85

[CONCLUSION] Overall: failed. The devmon repository cannot be successfully built on openEuler 24.03 LTS due to a C++ template compatibility issue between the libmcpp and mcengine subprojects. This is a code-level bug in the upstream subprojects that requires source code modification to resolve. The verification was performed without modifying any source code as required.

方案: Root cause: In libmcpp's table.h:725, the table_query template instantiation uses only object_type and IndexDef parameters, defaulting to local_storage_engine. However, mcengine's table objects use shm_storage_engine, causing a template type mismatch that cannot be resolved through build configuration options alone.. Recommendations: Report this issue to openUBMC/libmcpp upstream maintainers for fixing; Request libmcpp to fix table_query template to properly handle different storage engine types; Document in README that SQLite SQLITE_ENABLE_PREUPDATE_HOOK is required; Change subproject wrap files from SSH URLs to HTTPS URLs; Try building on Ubuntu 24.04 as recommended (may have different behavior)

来源: verification conclusion
