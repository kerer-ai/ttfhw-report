遇到的问题 (7)

HPCKit不支持 openEuler 24.03 LTS

方案: 修改 bootstrapper.conf 添加 'openEuler24.03(LTS)' 到 SUPPORT_OS_NAME 列表

来源: 主动适配 (patch HPCKit install script)

HMPI安装失败 - 'Unsupported os openEuler24.03(LTS)'

方案: 使用系统OpenMPI替代 (dnf install openmpi-devel)

来源: HMPI模块另有独立的OS检查

编译时找不到 mpi.h 头文件

方案: 添加 CPLUS_INCLUDE_PATH=/usr/include/openmpi-aarch64

来源: 系统OpenMPI安装路径与HMPI不同

链接时找不到 -lmpi 库

方案: 添加 LIBRARY_PATH=/usr/lib64/openmpi/lib

来源: 系统OpenMPI库路径不在默认搜索路径中

测试编译时 GCC mpicc 不支持 -stdlib=libstdc++ 标志

方案: 设置 OMPI_CC/OMPI_CXX 环境变量指向毕昇clang/clang++编译器

来源: test/functest/CMakeLists.txt 使用Clang特有编译选项

CMake FindThreads 在QEMU环境下-pthread检测失败

方案: 设置 -DCMAKE_THREAD_LIBS_INIT='-lpthread' -DTHREADS_PREFER_PTHREAD_FLAG=ON

来源: QEMU仿真限制

QEMU ARM64仿真下载速度极慢 (~30KB/s)

方案: 在宿主机(x86_64)下载HPCKit后通过docker cp传入容器

来源: QEMU user mode网络IO性能差
