文档缺失 (5)

#1安装文档仅提供 Ubuntu 系统的依赖安装命令，未提供 openEuler/RHEL 等其他 Linux 发行版的等效命令
#2文档未说明 LLVM 预编译包的平台兼容性问题（仅提供 Ubuntu x64 二进制，不适用于其他发行版）
#3CANN 版本与 Triton-Ascend 版本的对应关系复杂（商业版 vs 社区版），文档中表格较分散
#4文档未明确说明在无 NPU 硬件环境下的构建/测试替代方案
#5setup.py 构建流程自动下载 Ubuntu LLVM 包，未提供 openEuler/其他发行版的 LLVM 获取方式

遇到的问题 (2)

CMake 找不到 MLIR 包配置文件（MLIRConfig.cmake / mlir-config.cmake）

方案: 无法在合理时间内解决：构建系统自动下载的 LLVM 预编译包为 Ubuntu 构建，不包含 MLIR CMake 配置。需要从源码编译完整 LLVM 22 + MLIR（预计耗时1-2小时），且编译后的 MLIR 仍需 CANN 支持才能完成 Triton-Ascend 全功能构建

来源: CMakeLists.txt 第108行: find_package(MLIR REQUIRED)

LLVM Ubuntu 预编译包与 openEuler 系统库不兼容

方案: 需要从源码编译 LLVM 22 + MLIR for openEuler，或使用 openEuler 的 LLVM 包

来源: 构建系统自动下载 llvm-fad32722-afc55ca1-ubuntu-x64 到 ~/.triton/llvm/
