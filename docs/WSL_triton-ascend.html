遇到的问题 (4)

Docker 守护进程不可用，无法启动 openEuler 容器

方案: Docker CLI 已安装（v29.6.0），但守护进程无法启动（可能需要系统重启或配置 Hyper-V/WSL2）

来源: 宿主机环境限制

即使 Docker 可用，项目构建还需要 Ascend CANN Toolkit 9.0.0（华为专有软件）

方案: CANN Toolkit 需从华为官网 (https://www.hiascend.com) 下载，无法在自动化验证环境中获取

来源: docs/en/installation_guide.md - Installing CANN 章节

项目构建需要 LLVM 22 预编译包或从源码编译

方案: setup.py 自动从华为 OBS 下载预编译 LLVM，或需手动从源码编译（耗时极长）

来源: setup.py get_llvm_package_info() 函数

项目运行需要华为昇腾 NPU 硬件（Atlas A2/A3/950 系列）

方案: 无法在普通 x86 服务器上执行构建后的测试和样例，需要专用硬件环境

来源: README.md - Hardware Support 章节

documentation_gaps
  1. #1
    README.md 未直接列出构建依赖，需参考 docs/en/installation_guide.md 和 docker/Dockerfile
  2. #2
    docker/Dockerfile 使用 apt 包名（zlib1g-dev, libzstd-dev, clang-15, lld-15 等），需手动映射为 openEuler 包名（zlib-devel, libzstd-devel, clang, lld）

    installation_guide.md 中系统依赖安装示例仅给出 Ubuntu（apt）和 yum（zlib-devel）命令，未提供完整的 openEuler/dnf 安装指南

    README.md 未明确说明 LLVM 22 的具体版本要求，需参考 setup.py 中 cmake/llvm-hash.txt 和 third_party/ascend/llvm_patch/ 中的补丁文件
  5. #5
    setup.py 中 LLVM 预编译包下载地址为华为 OBS（triton-ascend-artifacts.obs.myhuaweicloud.com），在无网络环境中需手动准备
  6. #6
    CMakeLists.txt 要求 cmake >= 3.20，但 pyproject.toml 要求 cmake >= 3.20,< 4.0，版本上限一致
  7. #7
    文档未明确说明 AscendNPU-IR 子模块的编译需要大量磁盘空间和长时间编译
