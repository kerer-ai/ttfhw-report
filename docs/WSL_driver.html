#8drv_dvpp_cmdlist 内核模块 Makefile 引用 securec.h，该头文件依赖用户态 <stdarg.h>，在 kernel build 环境中需手动设置 CFLAGS_MODULE

遇到的问题 (5)

WSL2 内核版本（6.6.114.1-microsoft-standard-WSL2）与 openEuler 24.03 kernel-devel（6.6.0-145.oe2403）不匹配，默认内核模块编译路径无效

方案: 使用 build.sh -k 参数指定内核源码路径：-k /usr/src/kernels/6.6.0-145.0.13.139.oe2403.x86_64

来源: WSL2 容器环境限制

libascend_hal.so 链接失败：devdrv_hccl.c.o relocation R_X86_64_32S against .rodata，需重新编译为 -fPIC

方案: cmake 配置时添加 -DCMAKE_C_FLAGS=-fPIC，使 OBJECT library 生成位置无关代码

来源: src/ascend_hal/dms/drv_devmng/ascend910/CMakeLists.txt 中 devdrv_obj 未设置 POSITION_INDEPENDENT_CODE

drv_dvpp_cmdlist 内核模块编译失败：securec.h 包含 <stdarg.h> 用户态头文件，kernel build 环境无法找到

方案: 添加 CFLAGS_MODULE="-isystem $(gcc -print-file-name=include)" 使内核模块编译时可访问 GCC 内置头文件路径

来源: src/sdk_driver/dvpp/cmdlist/Makefile 第91行包含 -I${C_SEC_INCLUDE}，其中 securec.h 依赖 <stdarg.h>

样例无法编译运行：需要 CANN 软件包中的 dcmi 库

方案: 按 QUICKSTART 文档安装 CANN 软件包后，可使用 bash run.sh 命令运行样例

来源: examples/README.md 明确说明需先安装 CANN 软件包

make package 失败：package.py 找不到内核模块文件（未执行 make install）

方案: 在 make driver 之后执行 make install，将编译产物安装到 lib/ 目录，然后再执行 make package

来源: build.sh 中的 build_npu_driver 函数实际执行的是 make driver && make install
