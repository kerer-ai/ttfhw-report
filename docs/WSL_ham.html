遇到的问题 (5)

Docker 无法在本机运行: 本机为 WSL1 (内核 4.4.0-17763-Microsoft, Windows build 17763), 无真实 Linux 内核, dockerd 启动即 boltdb mmap panic (invalid page type: 0: 4); 且该 Windows 版本过旧 (需 18362+) 无法升级到 WSL2。

方案: 改为直接在 WSL 的 openEuler 24.03 (LTS-SP3) 发行版中验证。该发行版本身即为 skill 推荐的两个目标镜像之一, 与容器内环境同 OS/版本, 验证保真度高, 仅缺少容器隔离。

来源: skill 决策树步骤4 (Docker 不可用时通知用户); 用户确认按 skill 路径推进

src/CMakeLists.txt 链接 /usr/lib64/libboundscheck.so, 但该库默认不存在。

方案: 构建前先在 3rdparty/libboundscheck 执行 make 生成 libboundscheck.so, 复制到 /usr/lib64 并 ldconfig。

来源: src/CMakeLists.txt 第9-12行 + 3rdparty/libboundscheck/Makefile

build.sh / run_dt.sh 为 CRLF 行结尾, 在 Linux bash 报 $'\r' 语法错误。

方案: 对暂存副本执行 sed -i 's/\r$//' 规范化为 LF (未修改原始源码)。

来源: 源码在 Windows 端检出导致; 验证副本侧规范化

test/run_dt.sh 中 patch_mockcpp 调用 dos2unix (不可用) 并应用 arm64 专用补丁。

方案: 本机为 x86_64, 无需 arm64 补丁; 预先将 mockcpp_support_arm64.patch 放入 mockcpp 目录使 patch_mockcpp 提前 return 0, 跳过补丁与 dos2unix。

来源: test/run_dt.sh patch_mockcpp() 早返回逻辑

run_dt.sh 末尾的 lcov/genhtml 覆盖率步骤失败 (lcov 在 openEuler 24.03 默认源不可得)。

方案: 该步骤在全部测试通过且生成 gtest XML 之后执行, 不影响测试结论; 记录为覆盖率报告缺失。

来源: run_dt.sh 第44行 lcov: command not found

problems_encountered
  1. #1
    4 个字段
    timestamp
    problem
    Docker 无法在本机运行: 本机为 WSL1 (内核 4.4.0-17763-Microsoft, Windows build 17763), 无真实 Linux 内核, dockerd 启动即 boltdb mmap panic (invalid page type: 0: 4); 且该 Windows 版本过旧 (需 18362+) 无法升级到 WSL2。
    solution
    改为直接在 WSL 的 openEuler 24.03 (LTS-SP3) 发行版中验证。该发行版本身即为 skill 推荐的两个目标镜像之一, 与容器内环境同 OS/版本, 验证保真度高, 仅缺少容器隔离。
    source
    skill 决策树步骤4 (Docker 不可用时通知用户); 用户确认按 skill 路径推进
