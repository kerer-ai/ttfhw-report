遇到的问题 (3)

首次make build失败: No rule to make target '/usr/lib64/libboundscheck.so'

方案: 发现src/CMakeLists.txt硬编码了/usr/lib/lib64/libboundscheck.so路径。需先从3rdparty/libboundscheck子模块编译生成.so并安装到/usr/lib64/

来源: src/CMakeLists.txt第12行: /usr/lib64/libboundscheck.so

lcov在openEuler 24.03 LTS的dnf仓库中不可用

方案: lcov仅用于测试覆盖率报告，不影响核心构建和测试。尝试从GitHub下载但网络超时。标记为可选依赖缺失。

来源: dnf仓库搜索 + GitHub release下载尝试

pip3 install gcovr耗时过长

方案: 最终安装成功，但需要较长时间下载依赖

来源: 网络延迟
