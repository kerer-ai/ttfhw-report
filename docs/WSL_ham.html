遇到的问题 (4)

Docker 容器内 DNS 解析失败（getaddrinfo() thread failed to start）

方案: 使用 --network host 模式启动容器，并挂载宿主机 /etc/resolv.conf

来源: Docker 网络配置问题

首次构建因 CMakeCache.txt 路径不匹配失败

方案: 清理旧的 CMake 缓存文件（rm -rf build output CMakeCache.txt CMakeFiles）

来源: 宿主机构建残留缓存与容器内路径冲突

单元测试运行时 SIGILL (Illegal instruction)，6/8 测试崩溃

方案: 应用 test/3rdparty/mockcpp_support_arm64.patch 补丁，为 mockcpp 添加 aarch64 支持。UnixCodeModifier.cpp 的 2 个 hunk 需手动 sed 插入

来源: mockcpp 原生不支持 aarch64 架构，需社区补丁

pre-commit clang-tidy hook 失败，报错 'no checks enabled'

方案: clang-tidy 12.0.1 与 .clang-tidy 配置中的多行 YAML 格式不兼容，需升级 clang-tidy 或调整配置格式

来源: .clang-tidy 使用 YAML 多行语法 (>），clang-tidy 12.0.1 可能不完全支持
