架构
Rust workspace 项目（edition 2021），包含 ~20+ 子 crate。主二进制 stratovirt（轻量级 KVM hypervisor），附 stratovirt-img（镜像工具）。依赖 KVM 内核模块运行虚拟机，支持 microvm/q35/virt 机型。

文档缺失 (4)

#1Makefile yum-deps 目标使用 yum 命令（openEuler 24.03 已废弃），建议改为 dnf-deps 并使用 dnf 命令。
#2Cargo.lock 锁定 rand_core v0.10.0（edition2024），openEuler 24.03 官方 Rust 1.82.0 不支持，建议在 README 中明确最低 Rust 版本要求（>=1.85.0）。
#3stratovirt 作为 KVM hypervisor，集成测试完全依赖 /dev/kvm，无法在容器化 CI 中运行，建议提供 mock KVM 接口或使用裸金属/嵌套虚拟化环境。
#4tools/build_stratovirt_static/Dockerfile 基于 openeuler:22.03-lts-sp2（musl 静态编译），与当前 openEuler 24.03 LTS 环境不一致，README 中也未提及此静态构建方法。

遇到的问题 (3)

openEuler 24.03 LTS 官方 Rust 1.82.0 不支持 Cargo.lock 中 rand_core v0.10.0 需要的 edition2024 特性，cargo build 报错。

方案: 通过 USTC 镜像安装 rustup，升级到 Rust 1.96.0（2026-05-25 版本）。

来源: Cargo.lock → rand_core v0.10.0 requires edition2024

容器内无法直接访问 rustup.rs 官方站点和 crates.io，下载卡住超时。

方案: rustup-init.sh 在宿主机下载后 docker cp 传入；rustup 和 Cargo 均使用 USTC 镜像。

来源: 网络限制：容器到外网的连通性问题

Docker 容器内无 /dev/kvm 设备，19 个 mod_test 集成测试目标（600+ 测试用例）全部失败，且虚拟机启动示例无法执行。

方案: 这是容器环境的固有限制。所有单元测试（~430）全部通过，编译产物正确，冒烟测试成功。

来源: Docker 容器限制：无 /dev/kvm 设备透传

documentation_gaps
  1. #1
    Makefile yum-deps 目标使用 yum 命令（openEuler 24.03 已废弃），建议改为 dnf-deps 并使用 dnf 命令。
  2. #2
    Cargo.lock 锁定 rand_core v0.10.0（edition2024），openEuler 24.03 官方 Rust 1.82.0 不支持，建议在 README 中明确最低 Rust 版本要求（>=1.85.0）。
  3. #3
    stratovirt 作为 KVM hypervisor，集成测试完全依赖 /dev/kvm，无法在容器化 CI 中运行，建议提供 mock KVM 接口或使用裸金属/嵌套虚拟化环境。
  4. #4
    tools/build_stratovirt_static/Dockerfile 基于 openeuler:22.03-lts-sp2（musl 静态编译），与当前 openEuler 24.03 LTS 环境不一致，README 中也未提及此静态构建方法。
