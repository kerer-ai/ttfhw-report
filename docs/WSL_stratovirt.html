文档缺失 (5)

#1docs/build_guide.ch.md 声明 rustc 最低版本 1.64.0，但仓库 Cargo.lock 使用 v4 格式，实际需要 Rust >= 1.73.0
#2Makefile yum-deps 未包含 Dockerfile 中的 libcap-devel 和 libseccomp-devel，依赖列表在不同文件间不一致
#3docs/test.md 中 mod_test 需要外部仓库 virtiofsd（gitlab.com/virtio-fs/virtiofsd），但未提供备用获取方式
#4Dockerfile 使用 Docker Hub 镜像名 'openeuler/openeuler'，但 Docker Hub 在当前网络环境下不可达
#5README 中样例需要外部下载内核和 rootfs 镜像，未说明可在无 KVM 环境下使用 stratovirt-img 工具进行镜像操作来验证

遇到的问题 (3)

系统 Rust 1.64.0 无法解析 Cargo.lock v4 格式（需要 >= 1.73.0）

方案: 通过 rustup 安装 Rust 1.96.0 stable 工具链替代系统 Rust

来源: Cargo.lock 文件版本与文档声明的最低 Rust 版本不匹配

rustup 首次安装的工具链 cargo 组件缺失，导致 'cargo' binary is not applicable to the toolchain 错误

方案: 清除工具链目录后使用 --profile minimal 重新安装，cargo 组件正确安装

来源: rustup 组件管理元数据不一致（可能因之前中断的安装导致）

Dockerfile 指定的 Docker Hub 镜像（openeuler/openeuler:22.03-lts-sp2）不可达

方案: 改用 openEuler 官方仓库 hub.oepkgs.net/openeuler/openeuler:22.03-lts-sp2

来源: 网络环境限制
