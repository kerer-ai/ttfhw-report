文档缺失 (7)

#1README未明确列出独立编译xucg所需的系统包依赖 (仅提到yum install cmake flex)
#2README未提及hucx作为xucg的编译时前置依赖需要单独clone和编译 (仅在完整5步编译流程中隐含)
#3README将项目描述为支持openEuler AArch64, 但未在独立编译章节显式说明x86_64不支持
#4README.origin中的测试说明指向OSU benchmarks (需hmpi), 缺少独立单元测试运行说明
#5gatherv_sm.c 和 scatterv_sm.c 中硬编码ARM dmb内联汇编, 缺少 __aarch64__ 架构保护宏
#6tests/gtest/stub.cpp 中 ucg_global_params_t 结构体未初始化, 触发 GCC 12 -Wmaybe-uninitialized 警告 (因 -Werror 导致编译失败)
#7文档中未说明如何离线编译测试 (googletest需从GitHub下载, 但可设置 -DUCG_BUILD_WITH_GTEST 指向本地安装)

遇到的问题 (7)

WSL文件系统不支持符号链接, hucx克隆失败

方案: 使用 git -c core.symlinks=false clone 禁用符号链接

来源: Git文档 + 实验

hucx configure时缺少 infiniband/verbs.h 头文件

方案: dnf install -y rdma-core-devel libibverbs-devel

来源: 包搜索 + configure错误信息

xucg 编译失败: gatherv_sm.c 和 scatterv_sm.c 中ARM dmb内联汇编在x86_64上不合法

方案: 变通方案: 使用 gcc -E -P 预处理后 sed 替换 ARM dmb 指令为x86兼容代码。根本解决方案: 等待上游添加架构保护宏或使用AArch64环境

来源: 源码分析 + 架构差异

googletest无法从GitHub下载 (容器内无外网访问)

方案: 从openEuler仓库安装 gtest-devel, 创建符合CMake期望的目录结构, 设置 -DUCG_BUILD_WITH_GTEST=/tmp/gtest

来源: dnf包搜索 + CMakeLists.txt分析

test stub.cpp 中 ucg_global_params_t params 未初始化触发 -Werror=maybe-uninitialized

方案: 添加 -DCMAKE_CXX_FLAGS='-Wno-maybe-uninitialized' 编译选项

来源: GCC 12编译错误信息

117个测试因SM共享内存清理代码segfault (ARM dmb替换为空导致缺少内存屏障)

方案: 无法在x86_64上解决, 需在AArch64原生环境或完整aarch64交叉编译+QEMU环境下运行

来源: 架构差异 + 变通方案局限

无法交叉编译aarch64目标 (gcc-cross仅提供C编译器, 缺少C++编译器)

方案: openEuler仓库的gcc-cross包不完整, 需要手动安装完整aarch64工具链或使用其他发行版

来源: dnf包搜索
