文档缺失 (7)

#1仓库根目录 README 仅列出子组件链接，未提供统一的构建指南和依赖列表
#2virt-awaresched spec 文件 Requires 行包含 libvirt 和 libboundscheck，但 README 的「构建依赖」章节仅提到 patch, libvirt-devel, libboundscheck，未提及运行时依赖 libcap
#3ubs-virt-ovs spec 文件中 ninja-build 包名在 openEuler 24.03 上实际为 python3-ninja，存在包名不一致问题
#4virt-optimizer spec 文件和 build_rpm.sh 使用 Windows 行尾 (CRLF)，在 Linux 容器中执行 rpmbuild 会失败
#5ubs-virt-enpu/vcann-rt 的 make_build.sh 文件不存在于仓库中（仅有 CMakeLists.txt），文档中提到的构建脚本缺失
#6virt-optimizer 要求内核 eBPF 配置，但 README 未在「快速开始」章节明确提示此前置条件
#7各子组件的构建脚本 (build.sh) 使用 Windows 行尾，需先执行 dos2unix 转换

遇到的问题 (5)

shell 脚本使用 Windows 行尾 (CRLF)，在 Linux 容器中执行报错

方案: 安装 dos2unix 并批量转换所有 .sh 文件

来源: 仓库中的 build.sh 等脚本文件

ubs-virt-ovs rpmbuild 报错 ninja-build 包不存在

方案: 改用直接 cmake 编译（cmake .. && cmake --build .），绕过 rpmbuild 的依赖检查

来源: ubs-virt-ovs/build/ubs-virt-ovs.spec 中 BuildRequires: ninja-build，但 openEuler 24.03 中该包名为 python3-ninja

virt-optimizer RPM 打包失败，spec 文件包含 \r 字符

方案: cmake 编译本身成功，仅 RPM 打包受影响。需修复 spec 文件的行尾格式

来源: virt-optimizer/ubs_optimizer.spec 使用 Windows 行尾

ubs-virt-enpu 无法编译，缺少 CANN SDK 和 Ascend 硬件

方案: 该组件需要专用的昇腾 NPU 硬件环境，无法在通用 x86_64 环境中验证

来源: ubs-virt-enpu/vcann-rt/CMakeLists.txt 依赖 $ENV{ASCEND_HOME_PATH} 等环境变量

virt-awaresched 安全管理器测试失败（2个）

方案: 测试失败原因可能是容器环境中缺少 libcap 的 capability 操作权限，非代码缺陷

来源: test/vasd/security/test_vas_security_manager.cpp
