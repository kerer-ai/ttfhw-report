文档缺失 (8)

#1README未提及编译需要sqlite3命令行工具（仅在Makefile db target中隐式依赖）
#2README未说明Go版本要求（go.mod中声明go 1.14）
#3README中的golang-bin包名在openEuler 24.03中实际为golang包
#4README未说明构建时需要网络访问（Go模块虽使用vendor，但collector-install需git clone外部分仓库）
#5README未说明数据库初始化脚本(init.sql)对SQLite DQS(双引号字符串)的依赖
#6测试文档缺失：未说明集成测试需要systemd、已安装的atuned服务和真实硬件环境
#7README中hwloc-gui包在openEuler 24.03中实际安装hwloc包（无单独的gui包）
#8模型生成工具(generate_models.py)的perf依赖和权限要求未在README中记录

遇到的问题 (5)

make失败：db target SQLite初始化大量Parse error

方案: 未解决。需要修改init.sql将双引号改为单引号，或使用支持DQS=1的SQLite版本。

来源: 问题定位：SQLite compile_options输出显示DQS=0

集成测试无法在容器中运行

方案: 跳过。完整的集成测试需要在安装了A-Tune的openEuler物理机/虚拟机上运行。

来源: tests/test_lib.sh - check_service_started函数使用systemctl

模型生成工具(generate_models.py)执行perf失败

方案: 未解决。需要在特权容器（--privileged）或宿主机上运行。

来源: 问题定位：perf stat -a -e cycles返回exit status 255

atuned daemon无法启动（缺少配置）

方案: 执行make install即可解决。make install会安装配置文件、证书等。

来源: atuned.go:111 - could not find default config file

golang-bin包名在openEuler 24.03中不存在

方案: 已解决。自动安装了golang包（go 1.21.4）。

来源: dnf包搜索结果
