#6Dockerfile-atune-engine 基于 python:3.8-slim（Debian）而非 openEuler，无 openEuler 原生容器构建方案

遇到的问题 (5)

SQLite init.sql 双引号兼容性：database/init.sql 使用双引号包裹字符串值，SQLite 将其解析为列标识符而报 'no such column'

方案: 将 init.sql 中字符串值的双引号改为单引号

来源: database/init.sql

pip3 ARM64 编译超时：scipy/pandas/scikit-learn/xgboost 在 aarch64 上无预编译 wheel，需从源码编译，耗时超过 10 分钟

方案: 通过 dnf 安装系统 Python 包替代 pip 安装；或使用预编译的 aarch64 wheel

来源: pip3 install 命令

go test 无测试文件仍返回非零码：28 个 Go 包全部 [no test files]，vet 检查失败导致 exit code 1

方案: 修复 vet 警告（int64→string 转换、Printf 格式化），或使用 go test -vet=off

来源: go test ./...

atuned 守护进程不可用：Shell 集成测试需 systemd（容器内无 systemd），且须 root 完整安装 A-Tune 并启动三个服务

方案: 在具有 systemd 的环境中（物理机或 VM）进行完整安装验证；或使用 privileged 容器 + systemd

来源: tests/run_tests.sh + test_lib.sh

远程机器为 aarch64 架构：编译产物为 ARM64 ELF，与常见 x86_64 环境不兼容

方案: 注意跨架构部署需求，明确标注编译架构

来源: uname -m
