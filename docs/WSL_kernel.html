文档缺失 (4)

#1仓库未提供 Dockerfile 或容器化构建说明，外部开发者需自行搭建编译环境
#2Documentation/process/changes.rst 中的依赖包名与 openEuler 实际包名有差异（如 pkg-config → pkgconf），需手动适配
#3qemu/kvm 等虚拟化选项的构建文档不在 README 主入口直接可达，需浏览 Documentation/ 子目录
#4DTS (Device Tree Source) 编译在 arm64 defconfig 下部分失败，缺少 dt-bindings include 路径说明

遇到的问题 (4)

dnf install 时 perl-File-Basename 包不存在，导致整个事务回滚

方案: File::Basename 是 Perl 核心模块，随 perl 包自带，无需单独安装。移除该包名后重新安装成功。

来源: dnf 包名映射错误

make -j51 时 DTS 编译失败：dt-bindings 头文件（gpio.h, arm-gic.h, irq.h, input.h）找不到

方案: 这是因为 make 默认目标包含 dtbs 且并行构建时 DTS 编译器的 include 路径配置问题。改为 make -j51 Image modules 跳过 DTS 构建，内核核心编译成功。DTS 头文件实际存在于 include/dt-bindings/ 目录中。

来源: 内核构建系统 DTS include 路径配置

pip install pytest 默认源和清华镜像均不可用（超时/403）

方案: 切换至阿里云 PyPI 镜像 (mirrors.aliyun.com) 安装成功

来源: 容器内网络环境限制

make headers_install 失败：rsync 命令未找到

方案: dnf install -y rsync 安装后重试成功

来源: openEuler 基础镜像未预装 rsync
