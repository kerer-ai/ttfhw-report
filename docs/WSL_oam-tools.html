文档缺失 (6)

#1文档前置依赖要求 python >= 3.12.0，但 openEuler 24.03 LTS dnf 仓库仅提供 python3-3.11.6；实际 C++ 构建与全部 UT 在 3.11.6 下均可通过，3.12 要求偏保守且未说明 openEuler 环境下的获取方式
#2init_env.sh 的 setup_cann_env() 创建 pkg_inc/{mmpa,fmk,ts,adump} 符号链接这一关键步骤未在 quick_install.md 中说明，且该步骤在提前终止 init_env.sh 时会缺失，导致 cmake 报 mmpa_headers 路径不存在
#3build.sh 以 root 运行时默认 ASCEND_HOME_PATH=/usr/local/Ascend/latest，但 CANN toolkit 实际安装后该路径不存在（应为 ascend-toolkit/latest 或 cann），需用户显式设置 ASCEND_HOME_PATH，文档未提示
#4README 提到 pre-commit「首次需配置 java/maven 环境构建 jar 包」，但实际 scripts/oat_check.sh 已改为 Python 版 oat-py，文档描述与实现不一致
#5仓库无 Dockerfile，文档「Docker 部署」推荐的是华为预集成 CANN 的 Ubuntu 镜像，未提供基于 openEuler 的构建镜像参考
#6运行态样例（asys info/health/collect、msaicerr -e、msprof 采集）依赖 NPU 设备，文档未明确无 NPU 环境下可验证到何种程度

遇到的问题 (4)

首次构建 cmake 阶段报 git dubious ownership，FetchContent 克隆 cann-cmake 失败

方案: git config --global --add safe.directory '*'（WSL bind-mount 导致仓库目录属主与容器 root 不一致，属环境问题）

来源: build/_deps/cann-cmake-src（cmake/fetch_cann_cmake.cmake FetchContent）

第二次构建 cmake Generating 阶段报 mmpa_headers 目标引用不存在的 /usr/local/Ascend/cann/pkg_inc/mmpa

方案: 创建符号链接 ln -sf /usr/local/Ascend/cann/x86_64-linux/include/mmpa /usr/local/Ascend/cann/x86_64-linux/pkg_inc/mmpa（补全 init_env.sh setup_cann_env 被提前终止而缺失的步骤）

来源: src/msprof/collector/dvvp/msprofbin/CMakeLists.txt 与 acp/CMakeLists.txt 引用的 mmpa_headers 目标

pip install -r requirements.txt 首次因 files.pythonhosted.org ReadTimeoutError 失败

方案: 改用清华源 -i https://pypi.tuna.tsinghua.edu.cn/simple 并加 --timeout 180 --retries 8，torch 走 --extra-index-url pytorch cpu 索引，重试成功

来源: requirements.txt（torch==2.12.0+cpu 等大包）

init_env.sh 同时下载 toolkit(~790MB) 与 ops(~2GB) 后才安装，纯编译场景下 ops 下载耗时过长

方案: 提前终止 init_env.sh，直接用已下载的 toolkit .run 以 --quiet --full 安装；ops 为运行态依赖，文档明确仅编译可不安装

来源: init_env.sh install_cann() 顺序：先下完两个包再安装

documentation_gaps
  1. #1
    文档前置依赖要求 python >= 3.12.0，但 openEuler 24.03 LTS dnf 仓库仅提供 python3-3.11.6；实际 C++ 构建与全部 UT 在 3.11.6 下均可通过，3.12 要求偏保守且未说明 openEuler 环境下的获取方式
  2. #2
    init_env.sh 的 setup_cann_env() 创建 pkg_inc/{mmpa,fmk,ts,adump} 符号链接这一关键步骤未在 quick_install.md 中说明，且该步骤在提前终止 init_env.sh 时会缺失，导致 cmake 报 mmpa_headers 路径不存在
  3. #3
    build.sh 以 root 运行时默认 ASCEND_HOME_PATH=/usr/local/Ascend/latest，但 CANN toolkit 实际安装后该路径不存在（应为 ascend-toolkit/latest 或 cann），需用户显式设置 ASCEND_HOME_PATH，文档未提示
  4. #4
    README 提到 pre-commit「首次需配置 java/maven 环境构建 jar 包」，但实际 scripts/oat_check.sh 已改为 Python 版 oat-py，文档描述与实现不一致
  5. #5
    仓库无 Dockerfile，文档「Docker 部署」推荐的是华为预集成 CANN 的 Ubuntu 镜像，未提供基于 openEuler 的构建镜像参考
  6. #6
    运行态样例（asys info/health/collect、msaicerr -e、msprof 采集）依赖 NPU 设备，文档未明确无 NPU 环境下可验证到何种程度
