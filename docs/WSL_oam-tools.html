文档缺失 (7)

#1文档推荐的 AscendHub CANN 镜像(swr.cn-south-1.myhuaweicloud.com/ascendhub/cann:...)需 AscendHub 登录认证, 文档未说明公开拉取不可用, 外部开发者无账号无法获取

#3文档要求 python >= 3.12.0, 但 openEuler 24.03 LTS 仓库仅提供 python3.11.6; 实际构建与 UT 在 3.11 下可运行, 文档版本要求与默认镜像不匹配

遇到的问题 (5)

文档推荐的 AscendHub CANN 镜像公开拉取返回 not found(需登录认证)

方案: 改用默认 openEuler 24.03 LTS 基础镜像, 并通过仓库 init_env.sh 下载安装 CANN toolkit

来源: docs/zh/quick_install.md 方式2 + docker pull 实测

init_env.sh 安装 CANN toolkit 失败: .run 打印 EULA 后非交互式无法接受

方案: 手动执行 ./Ascend-cann-toolkit_8.5.0_linux-x86_64.run --full --install-path=/usr/local/Ascend --quiet(--quiet 即接受 EULA)

来源: .run --help 输出: 'Using this option means accepting the EULA'

pip 安装 requirements.txt 失败: ruff>=0.14.14 在默认 pypi 源 'from versions: none'

方案: 切换清华镜像: pip3 install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple --extra-index-url https://download.pytorch.org/whl/cpu

来源: pip index versions ruff -i 清华源 返回 0.14.14 可用

首次构建 cmake 阶段失败: git 'dubious ownership' 阻断 FetchContent 拉取 cann-cmake

方案: git config --global --add safe.directory '*' (容器内), 并清理 build/build_out 后重试

来源: build.log: fatal: detected dubious ownership in repository at '/workspace/build/_deps/cann-cmake-src'

install/upgrade/uninstall ST 用例失败: .run 安装报 ERR_NO:0x0095 安装目录父目录权限无效 + 无 NPU 设备

方案: 环境限制(无 NPU + 绑定挂载目录权限), 非代码缺陷; 记录为预期外环境失败, 不修改源码

来源: build/install_st_output.log / uninstall_st_output.log

documentation_gaps
  1. #1
    文档推荐的 AscendHub CANN 镜像(swr.cn-south-1.myhuaweicloud.com/ascendhub/cann:...)需 AscendHub 登录认证, 文档未说明公开拉取不可用, 外部开发者无账号无法获取
  2. #2
    init_env.sh 执行 CANN .run 安装时未传 --quiet 参数, 非交互式环境下 EULA 无法接受导致安装失败; 文档/脚本未提示该用法
  3. #3
    文档要求 python >= 3.12.0, 但 openEuler 24.03 LTS 仓库仅提供 python3.11.6; 实际构建与 UT 在 3.11 下可运行, 文档版本要求与默认镜像不匹配
  4. #4
    requirements.txt 中 ruff/torch 等包在默认 pypi.org 源索引查询失败(from versions: none), 文档未给出国内镜像配置建议; 需手动切换清华/阿里等镜像
  5. #5
    绑定挂载目录触发 git 'dubious ownership', 阻断 cmake FetchContent 拉取 cann-cmake; 文档/init_env.sh 未提示容器内需配置 git safe.directory
  6. #6
    install/upgrade/uninstall ST 用例依赖 NPU 设备与安装目录(非绑定挂载)权限, 文档未说明无 NPU 环境下这些用例的预期结果
  7. #7
    README 验证章节未单独列出 ST 用例(install/upgrade/uninstall)对 NPU 与安装态的硬性依赖, 易被误判为代码缺陷
