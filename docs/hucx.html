遇到的问题 (4)

首次configure未启用gtest，导致构建后无法运行单元测试

方案: 重新configure并添加--enable-gtest选项

来源: configure --help输出中发现--enable-gtest选项

编译报错infiniband/verbs.h: No such file or directory，缺少RDMA开发库头文件

方案: 安装libibverbs-devel和librdmacm-devel包

来源: README未列出此依赖，通过编译错误日志识别

gtest单元测试在CMA传输层测试中段错误崩溃，未完成全部测试

方案: 容器内无RDMA硬件，CMA（Cross Memory Access）传输层测试在容器环境中不兼容导致崩溃，属于环境限制而非代码缺陷

来源: gtest输出日志中的backtrace显示uct_cma_ep_tx_error触发崩溃

uct_hello_world样例在容器中运行失败，self传输层不支持

方案: 容器内无RDMA硬件和实际网络设备，self传输层不支持client/server模式，属于环境限制

来源: 样例输出'No supported (dev/tl) found'
