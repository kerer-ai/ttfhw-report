遇到的问题 (5)

HPCKit 无法自动下载 (需要鲲鹏社区注册登录)

方案: 使用 cmake 标志 -DUSE_KUPL=OFF -DUSE_KUPL_COMM=OFF -DUSE_KUPL_ALL=OFF 禁用 KUPL 依赖

来源: CMakeLists.txt 中 option(USE_KUPL ...) 支持关闭 + 源代码中使用 #ifdef USE_KUPL 守卫

mpi.h 找不到 - OpenMPI 头文件在 /usr/include/openmpi-aarch64/ 不在默认路径

方案: 设置环境变量 CPLUS_INCLUDE_PATH=/usr/include/openmpi-aarch64 和 LIBRARY_PATH=/usr/lib64/openmpi/lib

来源: 容器中 find /usr -name 'mpi.h' 查找

SVEBITS 在 KUNPENG_ARCH=native 时为空，导致 #if SVEBITS == 512 编译错误

方案: 使用 -DUSE_SVE=OFF 禁用 SVE 支持

来源: CMakeLists.txt 中 option(USE_SVE ...) 支持关闭

test/build.sh 只允许在鲲鹏 920 (CPU part 0xd22) 上编译测试程序

方案: 手动执行 mpicxx 编译命令绕过 CPU 检查

来源: test/build.sh 第8-13行的 CPU 检查逻辑

HMPI 不可用 (包含在 HPCKit 中)

方案: 使用 OpenMPI (dnf install openmpi openmpi-devel) 替代

来源: 容器 openEuler 24.03 软件源

problems_encountered
  1. #1
    3 个字段
    problem
    HPCKit 无法自动下载 (需要鲲鹏社区注册登录)
    solution
    使用 cmake 标志 -DUSE_KUPL=OFF -DUSE_KUPL_COMM=OFF -DUSE_KUPL_ALL=OFF 禁用 KUPL 依赖
    source
    CMakeLists.txt 中 option(USE_KUPL ...) 支持关闭 + 源代码中使用 #ifdef USE_KUPL 守卫
  2. #2
    3 个字段
    problem
    mpi.h 找不到 - OpenMPI 头文件在 /usr/include/openmpi-aarch64/ 不在默认路径
    solution
    设置环境变量 CPLUS_INCLUDE_PATH=/usr/include/openmpi-aarch64 和 LIBRARY_PATH=/usr/lib64/openmpi/lib
    source
    容器中 find /usr -name 'mpi.h' 查找
  3. #3

    solution

    source
    CMakeLists.txt 中 option(USE_SVE ...) 支持关闭
