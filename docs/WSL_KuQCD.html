文档缺失 (8)

#1HMPI 不支持 openEuler 24.03 LTS 基础版（仅支持 24.03 LTS-SP3），需手动提取并安装 HMPI 包
#2README 声称仅支持 openEuler 22.03 LTS-SP4，但 HPCKit 中实际上也有 openEuler 24.03 SP3 的 HMPI 包
#3test/build.sh 中有 CPU part 0xd22 (Kunpeng 920) 检测，非 Kunpeng 硬件上需手动编译测试
#4README 未说明 libnuma 运行时依赖（KUPL 间接依赖）
#5HPCKit 下载链接为通用页面，未提供直接 URL
#6MPI 运行时在容器中需要特殊配置 (--mca plm isolated)，文档未提及
#7README 中环境准备步骤假设已配置 yum 源和 module 工具，但未说明具体配置方法
#8build.sh 脚本过于简单，未控制并发数 (make -j 无限制)，可能在大核数机器上导致资源耗尽

遇到的问题 (5)

HMPI 安装失败：Unsupported os openEuler24.03(LTS)

方案: 手动提取 Hyper-MPI_26.0.RC1_aarch64_openEuler24.03SP3_Bisheng_hns-release.tar.gz 到 /opt/hpckit/HPCKit/26.0.RC1/hmpi/bisheng/

来源: HPCKit install.sh untar_HMPI() 函数 OS 检测

mpicxx wrapper 包含编译机器硬编码路径，无法使用

方案: 直接使用 clang++ 手动编译测试程序，手动指定 MPI include/lib 路径

来源: HMPI 二进制包预编译路径问题

链接时缺少 libnuma（KUPL 间接依赖）

方案: dnf install -y numactl-devel

来源: ld 链接错误

MPI 运行时在 Docker 容器中初始化失败 (plm_rsh_agent: ssh not found)

方案: 使用 mpirun --allow-run-as-root --mca plm isolated -np 1 强制本地 fork 模式

来源: OpenMPI 容器运行时限制

test/build.sh CPU 检测不通过 (非 Kunpeng 920 硬件)

方案: 手动编译测试程序，绕过 CPU part 0xd22 检测

来源: test/build.sh CPU 检测逻辑
