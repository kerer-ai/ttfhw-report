文档缺失 (4)

#1WSL 环境下克隆仓库必须使用原生 Linux 文件系统 (/home/...) 而非 Windows 文件系统 (/mnt/c/...)，否则 CRLF 转换会损坏 PNG 二进制文件，文档未说明此要求
#2构建并发度与内存限制未在文档中说明，默认并发可能触发 OOM (exit code 137)，需显式设置 --with-num-cores=4 和容器 --memory=12g
#3'file' 包依赖未在构建前置条件中列出（openEuler 容器默认未安装）
#4jtreg 测试框架未预装在容器镜像中，导致 16,727 个完整测试套件无法执行

遇到的问题 (4)

CRLF 转换损坏 PNG 二进制文件（构建失败）

方案: 在 WSL 原生 Linux 文件系统 (/home/weihaobin/tmp_src) 而非 Windows 文件系统 (/mnt/c/...) 中克隆仓库

来源: WSL interop 文件系统转换

默认并发度导致构建 OOM (exit code 137)

方案: 限制并发：--with-num-cores=4 和 JOBS=4，并设置 Docker 容器 --memory=12g

来源: build configure/make

缺少 'file' 包依赖

方案: 通过 dnf install -y file 安装

来源: openEuler 24.03 LTS 容器默认包集

dnf 安装时 GPG 检查失败

方案: 使用 --nogpgcheck 标志跳过 GPG 验证

来源: openEuler 容器 GPG 密钥配置
