遇到的问题 (7)

Docker Hub 网络不可达，无法拉取 multiarch/qemu-user-static 镜像

方案: 通过 apt 安装 qemu-user-binfmt 包 (已预装)，使用内核 binfmt_misc 机制支持 ARM 模拟

来源: apt package manager

容器中未预装 wget

方案: 通过 dnf install -y wget 安装

来源: openEuler dnf repository

autogen.sh 报告 'git: command not found' 警告

方案: 非致命警告。autoreconf 正常完成，SCM_BRANCH 设为 '<unknown>'，SCM_VERSION 设为 '0000000'，不影响编译

来源: configure.ac 第33-39行 (git 用于获取版本信息，非必需)

HPCKit 原始下载链接需要鲲鹏社区注册

方案: 使用用户提供的华为云镜像链接: https://mirrors.huaweicloud.com/kunpeng/archive/HPC/HPCKit/HPCKit_26.0.RC1_Linux-aarch64.tar.gz

来源: 用户手动提供

test_init_mt 报告内存池分配错误 (mm_recv_desc: Out of memory)

方案: 预期行为。容器环境中未配置 hugepages，共享内存传输尝试分配大页失败但不影响功能测试完成

来源: 容器环境限制

部分可选依赖不可用 (CUDA, ROCm, IB, Java, Go, GTEST, MPI, FUSE3, KNEM, XPMEM, GDR_COPY)

方案: 这些是可选的加速/绑定依赖，不影响核心通信库编译。仅启用基础传输层 (self, tcp, sysv, posix, cma)

来源: configure.ac + config/m4/*.m4

无法运行完整的 MPI 测试 (OSU benchmarks)

方案: OSU Micro Benchmarks 需要完整 HMPI 栈 (hucx + xucg + hmpi 全部编译安装) 和 mpicc 编译器。仅验证 hucx 仓库不足以运行

来源: README.md - 6. 测试方法 章节
