文档缺失 (4)

#1README中编译器模块版本示例为5.1.0，实际HPCKit 26.0.RC1中为5.1.0.2
#2README中测试运行路径为'[install_path]/gtest/kudnn_gtest'，但构建后需要从/workspace/out/test/dnn/llt/gtest/手动复制
#3README未说明KML BLAS库在容器中的LD_LIBRARY_PATH设置方式
#4pre_env_kudnn.sh中git clone GitHub可能因网络问题失败，README虽有fallback但未详细说明手动下载步骤

遇到的问题 (4)

编译器模块bisheng/compiler5.1.0/bishengmodule不存在

方案: 使用实际版本bisheng/compiler5.1.0.2/bishengmodule

来源: README中示例版本与实际HPCKit版本不一致

pre_env_kudnn.sh中git clone GitHub googletest超时卡住

方案: 改用wget下载googletest v1.15.2 release tarball后手动编译

来源: GitHub网络不通

kudnn_gtest运行时找不到libkblas.so.1

方案: 通过module load加载kblas/multi模块，由模块自动设置LD_LIBRARY_PATH

来源: KML库路径需由HPCKit模块管理

kudnn_gtest运行崩溃：Illegal instruction (SIGILL)

方案: 环境限制，无法修复。测试二进制使用-march=armv8.5-a+sve编译，但虚拟化鲲鹏920环境不支持SVE指令集

来源: 虚拟化环境CPU指令集限制
