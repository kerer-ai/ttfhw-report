文档缺失 (5)

#1仓库无 Dockerfile，文档推荐 Ubuntu 22.04 的 CANN 预装镜像，新开发者在 openEuler 上需手动安装 CANN toolkit
#2CANN toolkit 不在 openEuler dnf 仓库中，需从华为 DevCloud 镜像下载安装（1.5GB）
#3install_deps.sh 会从 GitHub 下载 googletest 源码编译，但 openEuler 的 dnf 直接提供 gtest/gtest-devel，脚本未优先检测系统包
#4python 建议版本 <= 3.10，但 openEuler 24.03 默认提供 python3.11（实测兼容）
#5pre-commit 配置文件使用了 tab 缩进（YAML 不合法），新开发者 clone 后 pre-commit 无法直接使用

遇到的问题 (2)

CANN toolkit 不在 openEuler dnf 仓库中

方案: 从华为 DevCloud 镜像下载 Ascend-cann-toolkit_9.1.0_linux-x86_64.run（1.5GB）并安装

来源: 外部资源（https://ascend.devcloud.huaweicloud.com）

.pre-commit-config.yaml 使用 tab 缩进，pre-commit 无法解析

方案: 需将 yaml 文件中的 tab 替换为空格才能使用 pre-commit，属于仓库文档缺陷

来源: 仓库根目录 .pre-commit-config.yaml
