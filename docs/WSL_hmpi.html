文档缺失 (8)

#1HPCKit 依赖：README 要求从鲲鹏社区下载 HPCKit，但该软件包为 AArch64 专属，x86_64 环境无法获取
#2hucx/xucg 依赖：README 完整构建流程依赖 hucx 和 xucg（需从 atomgit.com 克隆），但二者不在当前仓库中，且验证环境无法获取
#3OpenSHMEM 禁用方法未说明：平台文件 contrib/platform/mellanox/optimized 强制开启 OpenSHMEM，README 未说明如何在不支持 SPML 的环境下禁用
#4Fortran F08 绑定兼容性未说明：README 未说明哪些 GCC/gfortran 版本与 F08 绑定兼容
#5单元测试信息缺失：README 仅提供外部 OSU Micro Benchmark 测试方法，未说明内置 make check 的测试范围及限制
#6构建依赖清单不完整：README 仅列出 cmake 和 flex 两个依赖，未列出 autoconf/automake/libtool/m4/perl（仅在 HACKING 中提到）
#7无 Dockerfile：仓库不包含 Dockerfile 或容器化构建说明
#8构建并发数建议缺失：文档建议 make -j32 但未提供根据机器核数调整的建议

遇到的问题 (3)

configure 失败：OpenSHMEM 无法构建（无 SPML 传输层）

方案: 移除 --with-platform=contrib/platform/mellanox/optimized 参数（该平台文件第6行强制 enable_oshmem=yes），改用 --disable-oshmem 显式禁用

来源: contrib/platform/mellanox/optimized 第6行: enable_oshmem=yes

make 失败：Fortran F08 绑定编译错误 'Derived type mpi_datatype being used before it is defined'

方案: 添加 --enable-mpi-fortran=no 禁用 Fortran MPI 绑定（gfortran 12.3.1 与 OMPI 4.1.6rc4 的 F08 绑定存在已知兼容性问题）

来源: 编译器错误输出 + ./configure --help

mpirun 失败：plm_rsh_agent 找不到 ssh/rsh（容器内无可用的远程 Shell）

方案: 安装 openssh-clients + 使用 --mca plm_rsh_agent /bin/true 参数绕过 SSH 需求（单节点测试无需远程登录）

来源: Open MPI FAQ + mpirun 错误输出
