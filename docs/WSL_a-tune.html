文档缺失 (4)

#1README 中的 yum 包名在 openEuler 24.03 中部分已更名（如 hwloc-gui → hwloc, golang-bin → golang），需更新
#2init.sql 中的 INSERT 语句列名含特殊字符（句点、斜杠、花括号），在新版 sqlite3 中无法解析
#3未说明容器/Docker 环境下的测试运行方式，所有测试依赖 systemd 服务
#4Makefile 中 collector-install 依赖 git clone gitee.com，网络受限环境下会失败

遇到的问题 (3)

pip3 未预装，需先安装 python3-pip

方案: dnf install -y python3-pip

来源: 容器默认未安装 pip

pip install xgboost/scikit-learn 源码编译耗时过长

方案: 改用 dnf 安装预编译包 python3-xgboost python3-scikit-learn

来源: openEuler dnf 仓库

make db 目标失败：sqlite3 无法解析 init.sql 中含特殊字符的列名

方案: Go 代码编译不受影响，数据库初始化问题需修复 init.sql 中的列名转义

来源: database/init.sql
